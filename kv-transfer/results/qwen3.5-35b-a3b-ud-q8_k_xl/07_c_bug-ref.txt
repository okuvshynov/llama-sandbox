### 1. Critical Memory Leak & Double-Free in `table_delete`
**Location:** `table_delete` function, lines 58–61.
**The Bug:**
When deleting an entry, the code swaps the target entry with the last entry in the array:
```c
t->entries[i] = t->entries[t->size - 1];
```
This performs a **shallow copy** of the `Entry` struct.
1.  **Double-Free Risk:** The pointer to the string (e.g., `key` or `value`) from the last entry is now copied to position `i`.
