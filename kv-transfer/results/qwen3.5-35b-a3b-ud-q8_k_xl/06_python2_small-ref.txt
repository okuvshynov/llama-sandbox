### 2. `solve(edges)`

1. Build the graph

5. Backtrack to explore other possibilities

### 1. Program Overview

### 2. Function Details

#### `build_graph(edges)`
- Constructs a directed graph from a list of `(source, target)` edges.
- Tracks the in-degree (number of incoming edges) for each node.
- Ensures all nodes have an in-degree entry (defaulting to 0 if none exist).
- Returns:
  - `graph`: adjacency list
  - `in_degree`: dictionary of in-degree counts
  - `nodes`: set of all nodes

#### `solve(edges)`
- Implements **Kahn's algorithm** for topological sorting:
  1. Initialize a queue with all nodes having in-degree 0.
  2. Process each node from the queue:
     - Add it to the result.
     - Decrease the in-degree of its neighbors.
     - If a neighbor's in-degree becomes 0, add it to the queue.
  3. If the result contains all nodes, return the ordering. Otherwise, a cycle exists; return `None`.

- **Time Complexity:** O(V + E)  
- **Space Complexity:** O(V + E)

#### `find_all(edges)`
- Uses **recursive backtracking** to explore all possible topological orderings:
  1. Start with all nodes having in-degree 0.
  2. At each step, pick a valid node (in-degree 0, not yet in the path).
  3. Temporarily remove it and update neighbor in-degrees.
  4. Recurse to extend the path.
  5. When a complete ordering is found, save it.
  6. Backtrack to explore alternative paths.

- **Time Complexity:** O(V! × E) in worst case  
- **Space Complexity:** O(V²)
