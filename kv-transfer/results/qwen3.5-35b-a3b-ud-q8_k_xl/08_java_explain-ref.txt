# LRU Cache Implementation in Java

## Overview

This Java program implements a **Least Recently Used (LRU) Cache**, a data structure that stores key-value pairs up to a specified capacity and automatically removes the least recently used items when the limit is exceeded. It supports constant-time retrieval and updates.

## Data Structure

The LRU cache combines two core data structures:

### 1. **HashMap (`map`)**
- **Purpose:** Enables O(1) lookup of nodes by key.
- **Key-Value Mapping:** `K → Node<K, V>`

### 2. **Doubly Linked List**
- **Purpose:** Maintains item usage order.
- **Head:** Points to the most recently used item.
- **Tail:** Points to the least recently used item.
- **Dummy Nodes:** `head` and `tail` simplify insertion/deletion logic.

### 3. **Node Class**
```java
static class Node<K, V> {
    K key;
    V value;
    Node<K, V> prev, next;
}
```
Each node stores a key-value pair and references to its previous and next nodes in the list.

## How It Works

### `get(K key)`
- Retrieves the node using the HashMap.
- If found, removes it from its current position and moves it to the front (most recently used).
- Returns the associated value or `null` if not found.

### `put(K key, V value)`
- If the key exists:
  - Updates the value.
  - Moves the node to the front.
- If the key is new:
  - If the cache is full, removes the least recently used node (at `tail.prev`).
  - Adds the new node to the front.

### Helper Methods
- `remove(Node)` – Unlinks a node from the doubly linked list.
- `addToFront(Node)` – Inserts a node immediately after the head.
- `printState()` – Displays the cache contents in order from most to least recently used.

## Time and Space Complexity

| Operation | Time Complexity | Space Complexity |
|-----------|-----------------|------------------|
| `get()`   | O(1)            | O(capacity)      |
| `put()`   | O(1)            | O(capacity)      |

## Example Walkthrough

```java
LRUCache<String, Integer> cache = new LRUCache<>(3);
cache.put("a", 1);  // [a=1]
cache.put("b", 2);  // [b=2, a=1]
cache.put("c", 3);  // [c=3, b=2, a=1]
cache.printState(); // Output: Cache [c=3, b=2, a=1]

cache.get("a");     // Access 'a', moves to front
cache.printState(); // Output: Cache [a=1, c=3, b=2]

cache.put("d", 4);  // Cache full; evict 'b' (LRU)
cache.printState(); // Output: Cache [d=4, a=1, c=3]

cache.put("b", 5);  // 'b' is new; evict 'c' (LRU)
cache.printState(); // Output: Cache [b=5, d=4, a=1]
```

## Key Design Features

- **O(1) Operations:** Both `get()` and `put()` run in constant time.
- **Order Preservation:** The doubly linked list ensures correct LRU ordering.
- **Dummy Head/Tail Nodes:** Eliminate special handling for empty or single-element lists.
- **Type Generics:** Works with any key and value types (`K`, `V`).

## Use Cases

- **Web Caching:** Store frequently accessed pages or API responses.
- **Database Query Cache:** Keep recent query results.
- **Browser History:** Limit and manage recently visited pages.
- **Memory Management:** Maintain a bounded set of in-memory objects.

## Summary

This LRU Cache provides an efficient, type-safe mechanism for managing a fixed-size cache with fast access and automatic eviction of the least recently used items. It's a widely used pattern in systems where performance and memory constraints are critical.
