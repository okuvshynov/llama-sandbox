### 1. **Time Representation Mismatch (Critical)**
**Issue:**  
The `schedule()` method stores `time` as a relative delay (e.g., 1000ms from now), but the `run()` method treats `time` as an absolute timestamp. This causes incorrect wait calculations.

**Example:**
```javascript
schedule("backup", 1000, ...); // Stores 1000
// In run():
const wait = event.time - elapsed; // 1000 - elapsed → incorrect!
```

**Fix:**
Convert relative time to absolute timestamp when scheduling:
```javascript
schedule(name, delay, callback) {
  this.events.push({
    name,
    time: Date.now() + delay, // Absolute time
    callback
  });
  this.events.sort((a, b) => a.time - b.time);
}
```
Update the handler to log a human-readable execution time.

### 2. **Events Not Removed After Execution (Critical)**
**Issue:**  
The `run()` method never removes events after executing them. If `run()` is called again, the same events will execute repeatedly.

**Fix:**
Remove events after execution:
```javascript
for (const event of this.events) {
  // ...
  event.callback();
  // Remove executed event
  this.events.shift();
}
```
Alternatively, use `splice()` or rebuild the array after filtering.

---

### 3. **Async Handlers Not Awaited (Important)**
**Issue:**  
If a handler is async, it's not awaited, so the scheduler may proceed to the next event before the handler completes.

**Fix:**
Wait for each handler to resolve:
```javascript
for (const handler of handlers) {
  await handler(event);
}
```

---

### 4. **No Error Handling (Important)**
**Issue:**  
If a callback or handler throws an error, the entire scheduler stops without continuing to other events.

**Fix:**
Wrap callbacks and handlers in `try...catch`:
```javascript
try {
  event.callback();
} catch (err) {
  console.error(`Event ${event.name} failed:`, err);
}
```
Apply the same for handlers.

---

### 5. **Events Not Re-Sorted After Rescheduling (Medium)**
**Issue:**  
The `reschedule()` method updates an event's time but doesn't re-sort the `events` array. This can cause out-of-order execution.

**Fix:**
Re-sort after updating:
```javascript
reschedule(name, newTime) {
  for (const event of this.events) {
    if (event.name === name) {
      event.time = Date.now() + newTime; // or absolute time
    }
  }
  this.events.sort((a, b) => a.time - b.time);
}
```

### 6. **Inconsistent Equality Check (Minor)**
**Issue:**  
The `reschedule()` method uses loose equality (`==`) instead of strict equality (`===`), which can lead to unintended type coercion.

**Fix:**
Use strict equality:
```javascript
if (event.name === name) { ... }
```

### 7. **`run()` May Not Resolve on Error (Minor)**
**Issue:**  
If an unhandled error occurs during event execution, the Promise returned by `run()` may hang or fail silently.

**Fix:**
Ensure the Promise resolves even if errors occur, using `finally()` or `catch()`:
```javascript
run().catch(err => console.error("Scheduler error:", err));
```

### 8. **Handler Receives Outdated `event.time` (Medium)**
**Issue:**  
The handler logs `e.time`, which reflects the original relative time, not the actual time of execution.

**Fix:**
Pass the actual execution time to the handler:
```javascript
const executionTime = Date.now();
event.callback();
handler({ ...event, executionTime });
```
