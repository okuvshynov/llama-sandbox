This C++ program implements **Huffman coding**, a lossless data compression algorithm that assigns variable-length binary codes to characters based on their frequency in the input text. More frequent characters receive shorter codes, while less frequent ones receive longer codes, resulting in an efficient encoding scheme.

---

### Program Overview

1. **Node Structure:**
   - Each node contains a character (`ch`), frequency (`freq`), and pointers to left and right children.
   - Leaf nodes represent individual characters; internal nodes represent merged subtrees.

2. **Priority Queue:**
   - Uses a min-heap to efficiently retrieve and merge the two nodes with the lowest frequencies.

3. **Core Functions:**
   - `buildTree()`: Constructs the Huffman tree by combining the lowest-frequency nodes until one root remains.
   - `buildCodes()`: Traverses the tree to assign binary codes (left = '0', right = '1') to each character.
   - `encode()`: Translates the input text into a binary string using the generated codes.
   - `main()`: Orchestrates the process, displays the codes, and compares original vs. compressed sizes.

4. **Example Output:**
   - Displays each character's Huffman code, sorted by code length.
   - Reports the original size (in bits), encoded size, and compression ratio.

---

### Time Complexity Analysis

Let:
- `n` = length of the input text
- `k` = number of unique characters (typically ≤ 256 for ASCII)

| Function             | Complexity               | Explanation                                         |
|----------------------|--------------------------|-----------------------------------------------------|
| `buildTree()`        | O(n + k log k)           | Frequency counting (O(n)) + k insertions/deletions from heap (O(k log k)) |
| `buildCodes()`       | O(k)                     | Tree traversal over k nodes                         |
| `encode()`           | O(n)                     | One lookup per character                            |
| `main()`             | O(k log k)               | Sorting codes by length                             |
| **Total**            | **O(n + k log k)**       | Dominated by frequency counting and encoding        |

Since `k` is small and bounded, the overall time complexity is effectively **O(n)** in practice.

---

### Space Complexity

| Component            | Space              | Notes                                             |
|----------------------|--------------------|---------------------------------------------------|
| Frequency map        | O(k)               | Stores counts for each unique character           |
| Priority queue       | O(k)               | Holds tree nodes during construction              |
| Huffman tree         | O(k)               | k leaf nodes + k-1 internal nodes                 |
| Codes map            | O(k)               | Stores binary string for each character           |
| Encoded output       | O(n × avg code len)| Typically less than n bytes (compression effect)  |
| **Total**            | **O(n + k)**       | Linear in input size                              |

- **Purpose:** Compress text using frequency-based binary codes.

- **Space Complexity:** O(n + k)
- **Key Advantages:** Lossless compression, efficient for repetitive data
- **Limitations:** Requires storing the code table; less effective on already compressed data
