This Java program implements a **Least Recently Used (LRU) Cache**.

An LRU cache is a data structure that stores a fixed number of items. When the cache reaches its capacity and a new item needs to be added, it automatically removes the item that hasn't been accessed (read or written) for the longest time.

### 1. What the Program Does

The program defines a generic class `LRUCache<K, V>` that supports two primary operations in **O(1)** (constant) time complexity:
*   **`get(key)`**: Retrieves the value associated with a key. If the key exists, it marks the item as "recently used" by moving it to the front of the list. If the key doesn't exist, it returns `null`.
*   **`put(key, value)`**: Inserts or updates a key-value pair.
    *   If the key exists, it updates the value and moves the item to the front.
    *   If the key is new and the cache is full, it evicts the **least recently used** item (the one at the very end of the list) before inserting the new one.
    *   If the key is new and there is space, it simply adds the item to the front.

#### Walkthrough of the `main` method logic:
1.  **Initialization**: Creates a cache with a capacity of **3**.
2.  **Insertion**: Adds `a=1`, `b=2`, `c=3`.
    *   *State*: `[c=3, b=2, a=1]` (Newest on left, Oldest on right).
3.  **Access**: Calls `get("a")`.
    *   Since `a` was the oldest, accessing it moves it to the front.
    *   *State*: `[a=1, c=3, b=2]`.
4.  **Insertion (Eviction)**: Calls `put("d", 4)`.
    *   Cache is full (size 3). The least recently used item is now `b` (at the tail).
    *   `b` is removed. `d` is added to the front.
    *   *State*: `[d=4, a=1, c=3]`.
5.  **Update (Eviction)**: Calls `put("b", 5)`.
    *   Cache is full. The least recently used item is now `c`.
    *   `c` is removed. `b` is added to the front with the new value.
    *   *State*: `[b=5, d=4, a=1]`.

### 2. Data Structure Description

To achieve **O(1)** time complexity for both `get` and `put` operations, this implementation uses a hybrid data structure combining a **Hash Map** and a **Doubly Linked List**.

#### Components:

1.  **HashMap (`map`)**:
    *   **Type**: `Map<K, Node<K, V>>`
    *   **Purpose**: Provides fast lookup. It maps the `key` directly to the `Node` object in the linked list.
    *   **Role**: Allows the program to find any node instantly without traversing the list.

2.  **Doubly Linked List**:
    *   **Structure**: Custom `Node` class containing `key`, `value`, `prev`, and `next` pointers.
    *   **Sentinel Nodes**: Uses dummy `head` and `tail` nodes to simplify edge cases (adding/removing from empty lists or ends).
        *   `head.next` points to the **Most Recently Used (MRU)** item.
        *   `tail.prev` points to the **Least Recently Used (LRU)** item.
    *   **Role**: Maintains the order of usage.
        *   **Front (near head)**: Most recently accessed items.
        *   **Back (near tail)**: Least recently accessed items (candidates for eviction).

#### How they work together:

*   **Reading (`get`)**:
    1.  Look up the key in the **HashMap** to get the `Node` (O(1)).
    2.  Use the `Node`'s pointers to unlink it from its current position in the **Linked List** (O(1)).
    3.  Re-insert the `Node` right after the `head` (O(1)).

*   **Writing (`put`)**:
    1.  Check **HashMap** for existing key.
    2.  If capacity is exceeded, identify the LRU node via `tail.prev`, remove it from the **Linked List**, and remove its key from the **HashMap** (O(1)).
    3.  Create a new `Node`, add it to the **HashMap**, and insert it at the front of the **Linked List** (O(1)).

### Summary of Complexity

| Operation | Time Complexity | Space Complexity |
| :--- | :--- | :--- |
| **Get** | O(1) | O(capacity) |
| **Put** | O(1) | O(capacity) |

This combination is the standard optimal solution for implementing an LRU cache because neither a standalone Hash Map (which has no order) nor a standalone Linked List (which requires O(N) search) could achieve O(1) for both operations simultaneously.
