This Java program implements a **Least Recently Used (LRU) Cache**. An LRU cache is a data structure that stores a fixed number of items and automatically removes the least recently accessed item when the cache reaches its capacity limit.

### 1. What the Program Does

The program provides a generic cache (`LRUCache<K, V>`) that supports two main operations with **O(1)** time complexity:
*   **`get(key)`**: Retrieves the value associated with a key. If the key exists, it marks the item as "recently used" by moving it to the front of the list. If the key doesn't exist, it returns `null`.
*   **`put(key, value)`**: Inserts or updates a key-value pair.
    *   If the key already exists, it updates the value and moves the item to the front.
    *   If the key is new and the cache is full, it evicts the **least recently used** item (the one at the very end of the list) before inserting the new item at the front.

#### Walkthrough of the `main` method logic:
1.  **Initialization**: Creates a cache with a capacity of **3**.
2.  **Insertion**: Adds `a=1`, `b=2`, `c=3`. The order (most recent to least) is `c, b, a`.
3.  **Access**: Calls `get("a")`. Since `a` was accessed, it moves to the front. Order becomes `a, c, b`.
4.  **Eviction**: Calls `put("d", 4)`. The cache is full (3 items). The least recently used item is `b` (at the tail). `b` is removed, and `d` is added to the front. Order becomes `d, a, c`.
5.  **Update & Eviction**: Calls `put("b", 5)`. The cache is full again. The least recently used item is now `c`. `c` is removed, and `b` is added to the front. Order becomes `b, d, a`.

---

### 2. Data Structure Description

The implementation uses a hybrid data structure combining a **Hash Map** and a **Doubly Linked List**. This combination is the standard approach to achieve O(1) performance for both access and eviction.
