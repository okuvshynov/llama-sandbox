This Python program solves the **Topological Sorting** problem for a directed graph. Specifically, it models a set of tasks with dependencies (like a build pipeline) and determines valid orders in which these tasks can be executed.

Here is a breakdown of what the program does and the algorithms it employs.

### 1. Program Functionality
The program simulates a software build process where certain steps must happen before others.
- **Input**: A list of tuples representing directed edges `(u, v)`, meaning task `u` must be completed before task `v` can start.
- **Output**:
  1. **`solve(edges)`**: Finds **one** valid execution order. If a cycle exists (making execution impossible), it returns `None`.
  2. **`find_all(edges)`**: Finds **every possible** valid execution order.

In the provided example, the tasks represent a CI/CD pipeline:
- `deps` and `lint` have no prerequisites.
- `compile` depends on `lint` and `deps`.
- `link` and `test` depend on `compile`.
- `package` depends on both `link` and `test`.
