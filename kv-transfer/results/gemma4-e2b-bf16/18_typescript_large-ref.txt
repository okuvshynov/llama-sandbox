## 1. Security Vulnerabilities (Critical)

### Issue 1. SQL Injection in Repository Layer
The code uses string concatenation to build SQL queries with user-provided input, which makes the application highly susceptible to SQL injection attacks.

**Locations:**
1.  `UserRepository.findByEmail(email: string)`
    ```typescript
    // VULNERABLE CODE
    const result = await pool.query(
      "SELECT * FROM users WHERE email = '" + email + "'"
    );
    ```
2.  `AuditLogRepository.findByUser(userId: number, startDate: string, endDate: string)`
    ```typescript
    // VULNERABLE CODE
    const result = await pool.query(
      `SELECT * FROM audit_log WHERE user_id = $1 AND timestamp BETWEEN '${startDate}' AND '${endDate}' ORDER BY timestamp DESC`,
      [userId]
    );
    ```

**Fix:**
**Always use parameterized queries** provided by the `pg` library (`$1`, `$2`, etc.) for *all* dynamic values.

```typescript
// Fixed UserRepository.findByEmail
async findByEmail(email: string): Promise<User | null> {
  const result = await pool.query("SELECT * FROM users WHERE email = $1", [email]);
  return result.rows[0] || null;
}

// Fixed AuditLogRepository.findByUser
async findByUser(userId: number, startDate: string, endDate: string): Promise<AuditLogEntry[]> {
  // Ensure dates are correctly parameterized, even if they are simple strings.
  const query = `SELECT * FROM audit_log WHERE user_id = $1 AND timestamp BETWEEN $2 AND $3 ORDER BY timestamp DESC`;
  const result = await pool.query(query, [userId, startDate, endDate]);
  return result.rows;
}
```

### Issue 2. Weak Password Hashing (Critical)
The `AuthService` uses a custom, weak hashing function (`sha256` with a salt) instead of industry-standard, computationally expensive, and salted hashing algorithms like **Bcrypt** or **Argon2**.

**Location:** `AuthService.register` and `AuthService.login`

**Fix:**
Integrate a robust hashing library (like `bcrypt`).

```typescript
// Requires installing 'bcrypt'
import * as bcrypt from 'bcrypt';

// Inside AuthService.register
async register(email: string, username: string, password: string, role: string): Promise<{ user: User; token: string }> {
  // ... existing checks ...
  const salt = await bcrypt.genSalt(10); // 10 rounds is a good default
  const passwordHash = await bcrypt.hash(password, salt); // Use bcrypt hash
  const user = await this.userRepo.create(email, username, passwordHash, role);
  // ...
}

// Inside AuthService.login
async login(email: string, password: string): Promise<{ user: User; token: string }> {
  const user = await this.userRepo.findByEmail(email);
  if (!user) {
    throw new Error("Invalid credentials");
  }
  // Compare the provided password against the stored hash
  const match = await bcrypt.compare(password, user.passwordHash);
  if (!match) {
    throw new Error("Invalid credentials");
  }
  // ...
}
```

### Issue 3. JWT Secret Management
The JWT secret is hardcoded as a default value.

**Location:** `config.jwtSecret: process.env.JWT_SECRET || "default-secret-change-me"`

**Fix:**
Ensure the environment variable is strictly required and fails startup if missing.

```typescript
// Configuration check (in initialization or config file):
const jwtSecret = process.env.JWT_SECRET;
if (!jwtSecret) {
  throw new Error("FATAL: JWT_SECRET environment variable is not set.");
}
```
