## 1. Critical Memory Leak Bug (in `table_delete`)

The function `table_delete` fails to free the memory associated with the entry being deleted before overwriting the structure.

### Location of the Bug: `table_delete`

```c
void table_delete(Table* t, const char* key) {
    for (int i = 0; i < t->size; i++) {
        if (strcmp(t->entries[i].key, key) == 0) {
            // CRITICAL ISSUE: The memory pointed to by t->entries[i].key 
            // and t->entries[i].value is lost/leaked here.
            t->entries[i] = t->entries[t->size - 1];
            t->size--;
            return;
        }
    }
}
```

**Explanation:**

When an entry is found and replaced by copying the last element (`t->entries[i] = t->entries[t->size - 1];`), the original dynamically allocated `key` and `value` strings stored in `t->entries[i]` are no longer referenced by any part of the structure. Since they are not freed, they result in a **memory leak**.

**Fix for `table_delete`:**

Before overwriting the entry, you must free the old dynamic memory:

```c
void table_delete(Table* t, const char* key) {
    for (int i = 0; i < t->size; i++) {
        if (strcmp(t->entries[i].key, key) == 0) {
            // 1. Free the memory of the entry being deleted
            free(t->entries[i].key);
            free(t->entries[i].value);

            // 2. Replace the entry (copying the last element)
            t->entries[i] = t->entries[t->size - 1];
            t->size--;
            return;
        }
    }
}
```
