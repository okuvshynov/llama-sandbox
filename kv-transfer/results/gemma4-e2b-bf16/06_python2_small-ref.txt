## 1. Function Breakdown

### `build_graph(edges)`

**Purpose:** To transform the list of edges into a usable graph representation (adjacency list) and calculate the in-degree for every node.

**Mechanism:**
1. **Initialization:** It initializes an adjacency list (`graph`), a dictionary to store in-degrees (`in_degree`), and a set to track all unique nodes (`nodes`).
2. **Processing Edges:** It iterates through every edge $(u, v)$:
    * It adds $v$ to the list of neighbors of $u$ in the `graph`.
    * It increments the in-degree count for node $v$ (since $v$ has an incoming edge from $u$).
    * It records both $u$ and $v$ as nodes.
3. **Finalization:** It ensures that any node present in the graph structure but without an incoming edge is initialized with an in-degree of 0.

**Output:** The function returns the graph structure, the in-degree map, and the set of all nodes.

### `solve(edges)`

**Purpose:** To find **one** valid topological ordering of the graph.

**Algorithm:** This function implements **Kahn's Algorithm** for topological sorting.

1. **Initialization:**
    * It first builds the graph and in-degrees.
    * It initializes a queue (`deque`) with all nodes that have an initial in-degree of 0 (these are the starting tasks with no dependencies).
2. **Processing (BFS):**
    * While the queue is not empty, it dequeues a node.
    * This node is added to the `result` list (the topological order).
    * For every neighbor of the dequeued node:
        * Its in-degree is decremented (because the dependency on the current node has been satisfied).
        * If a neighbor's in-degree drops to 0, that neighbor is added to the queue, as it is now ready to be processed.
3. **Cycle Detection:** After the process is complete, it checks if the length of the resulting order matches the total number of nodes.
    * If the lengths are equal, a valid topological sort was found, and it returns the result.
    * If the lengths are unequal, it means the graph contains a **cycle** (a deadlock), and no topological sort is possible, so it returns `None`.

### `find_all(edges)`

**Purpose:** To find **all possible** valid topological orderings of the graph.

**Algorithm:** This function uses a **Backtracking Search** approach.

1. **Initialization:** It builds the graph and in-degrees. It uses a recursive helper function, `backtrack`.
2. **Backtracking Logic (`backtrack(path, remaining_in_degree)`):**
    * **Base Case:** If the current path contains all nodes (`len(path) == len(nodes)`), a complete valid order has been found. It is added to the `results` list.
    * **Iteration:** It iterates through all available nodes (sorted for consistent output, though not strictly necessary for correctness).
    * **Constraint Check:** A node is considered only if:
        a. It has not been included in the current `path`.
        b. Its `remaining_in_degree` is currently 0 (meaning all its prerequisites have been satisfied).
    * **Action (Choose):** If the node is valid:
        * It is added to the current `path`.
        * A *copy* of the current in-degrees is created (`new_degrees`) to preserve the state for backtracking.
        * The in-degrees of all neighbors of the chosen node are decremented in this copy.
        * The function recursively calls itself with the updated path and new degrees.
    * **Backtrack (Unchoose):** After the recursive call returns, the chosen node is removed from the current `path` (`path.pop()`), allowing the search to explore other possibilities.

**Output:** A list containing every unique list representing a valid topological ordering.
