## 1. Concurrency and Race Conditions

### Problem 1: Unreliable Queue Size Reporting in `/stats`

The `/stats` handler attempts to report the current size of the job queue using `len(pool.jobQueue)`.

**Issue:** For buffered channels in Go, the `len()` function does not reliably return the current number of elements in the queue. It typically returns the capacity, or an implementation-dependent value, but it is not a safe, accurate real-time count of waiting jobs.

**Impact:** The reported `queue_size` statistic is likely incorrect and misleading.

**Fix:** Since the `jobQueue` is a channel, the only reliable way to track the current number of jobs waiting is to introduce a dedicated counter protected by a mutex within the `WorkerPool` structure.
