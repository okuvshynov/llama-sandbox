## 1. Program Explanation

The program performs the following steps:

### A. Data Structures
1. **`Node`**: Represents a node in the Huffman tree. It stores a character (`ch`), its frequency (`freq`), and pointers to its left and right children.
2. **`Compare`**: A custom comparator used by the priority queue. It ensures that the node with the *lowest* frequency has the highest priority (a min-heap based on frequency).

### B. `buildTree(const std::string& text)`
This function constructs the Huffman tree:
1. **Frequency Calculation:** It first iterates through the input `text` and calculates the frequency of every unique character, storing them in an `unordered_map`.
2. **Initialization:** It creates a `priority_queue` (min-heap) and pushes a `Node` for every unique character found in the text.
3. **Tree Construction:** It iteratively extracts the two nodes with the lowest frequencies from the priority queue, creates a new internal parent node whose frequency is the sum of the two children's frequencies, and inserts this new parent back into the queue.
4. **Result:** This process continues until only one node remains—the root of the complete Huffman tree.

### C. `buildCodes(Node* node, ...)`
This function traverses the completed Huffman tree to generate the binary codes:
1. **Traversal:** It uses a Depth First Search (DFS) approach recursively.
2. **Code Assignment:**
    *   When it reaches a leaf node (a node with no children), it records the accumulated path (prefix) as the character's code.
    *   Moving to the left child appends a '0' to the prefix.
    *   Moving to the right child appends a '1' to the prefix.
3. **Result:** It populates an `unordered_map` where the key is the character and the value is its corresponding binary code string (e.g., `'a' \to "01"`, `'b' \to "101"`).

### D. `encode(const std::string& text, ...)`
This function performs the final encoding:
1. It iterates through the original input `text`.
2. For each character, it looks up its corresponding variable-length binary code in the generated `codes` map.
3. It concatenates these binary codes to form the final compressed `encoded` string.

### E. `main()`
The main function orchestrates the process:
1. It defines the input text.
2. It calls `buildTree` to create the optimal prefix code structure.
3. It calls `buildCodes` to determine the actual codes.
4. It sorts the character codes based on their length (shortest codes first), which is standard practice for displaying Huffman tables.
5. It prints the resulting codes.
6. It performs the final encoding and calculates and displays the compression ratio.
