## 1. Core Data Structures and Function Definitions

The program defines four main functions: `compress`, `decompress`, `analyze`, and `main`.

### `fn compress(input: &[u8]) -> Vec<(u8, u16)>`
**Purpose:** Implements Run-Length Encoding (RLE). This function takes a sequence of bytes and compresses it by replacing consecutive sequences of the same byte with a pair: `(byte value, count)`.

**Step-by-step logic:**
1. **Initialization:** It initializes an empty `Vec` (`result`) to store the compressed pairs.
2. **Edge Case:** If the input slice is empty, it returns the empty result vector immediately.
3. **Iteration:** It initializes `current` to the first byte and `count` to 1. It then iterates through the rest of the input bytes.
4. **Grouping:**
    * If the current byte matches the `current` byte, it increments the `count` (as long as `count` doesn't exceed the maximum value for `u16`).
    * If the byte changes (or the count overflows), it signifies the end of a run. The function pushes the finished run `(current, count)` to the `result`, and then resets `current` to the new byte and `count` to 1.
5. **Final Run:** After the loop finishes, the last run of bytes must be recorded, so it explicitly pushes the final `(current, count)` pair.
6. **Return:** Returns the vector of compressed pairs.

### `fn decompress(data: &[(u8, u16)]) -> Vec<u8>`
**Purpose:** Reverses the compression process. It takes the compressed data (a list of `(byte, count)` pairs) and reconstructs the original byte sequence.

**Step-by-step logic:**
1. **Initialization:** It initializes an empty `Vec` (`result`) to store the decompressed bytes.
2. **Iterate Compressed Data:** It loops through every tuple `(byte, count)` in the input `data`.
3. **Reconstruction:** For each tuple, it loops `count` number of times, pushing the specific `byte` into the `result` vector during each iteration.
4. **Return:** Returns the final, fully reconstructed vector of bytes.

### `fn analyze(input: &[u8]) -> HashMap<u8, usize>`
**Purpose:** Calculates the frequency (count) of every unique byte in the input data.

**Step-by-step logic:**
1. **Initialization:** It creates an empty `HashMap` named `freq`.
2. **Counting:** It iterates through every byte (`b`) in the input slice.
3. **Insertion/Update:** For each byte, it uses the `entry()` method. If the byte is already in the map, it increments its existing count; if it's new, it inserts the byte with an initial count of 1.
4. **Return:** Returns the map where the key is the byte value (`u8`) and the value is its frequency (`usize`).
