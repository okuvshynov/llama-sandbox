## 🧠 Algorithm Descriptions

### 1. `build_graph` (Graph Construction)

This function is a preprocessing step used by both main routines.

*   **Purpose:** To convert the list of dependency edges (tuples) into usable data structures:
    1.  **Adjacency List (`graph`):** Stores for each node $u$, a list of all nodes $v$ that depend on $u$ (i.e., $u \to v$).
    2.  **In-Degree Map (`in_degree`):** Stores, for every node $v$, the number of incoming edges. This count tells us how many prerequisites a node has.
    3.  **Node Set (`nodes`):** A list of all unique entities in the graph.
*   **Time Complexity:** $O(V + E)$, where $V$ is the number of nodes and $E$ is the number of edges.

### 2. `solve` (Finding One Topological Order)

This function implements **Kahn's Algorithm**, a widely used and highly efficient method for topological sorting, based on Breadth-First Search (BFS).

*   **How it Works (Kahn's Algorithm):**
    1.  **Initialization:** Start by identifying all nodes that have an in-degree of 0. These are the nodes with no prerequisites and can be executed first. These starting nodes are placed in a queue.
    2.  **Traversal:** While the queue is not empty:
        a. Dequeue a node and add it to the `result` list.
        b. For every neighbor (dependent task) of the dequeued node:
            i. Decrement the neighbor's in-degree (because one of its prerequisites has just been satisfied).
            ii. If the neighbor's in-degree drops to 0, it means all of its prerequisites are now met, so it is enqueued.
    3.  **Cycle Detection:** If the final `result` list contains fewer nodes than the total number of nodes in the graph, it means some nodes were never able to reach an in-degree of 0. This implies there is a cycle in the graph (e.g., A depends on B, and B depends on A), and no valid order exists.

*   **Time Complexity:** $O(V + E)$. This is very fast.

### 3. `find_all` (Finding All Topological Orders)

This function uses a **Backtracking Search** algorithm to explore every possible valid sequence. Since there can be multiple nodes available at the same time (i.e., multiple nodes with an in-degree of 0), the algorithm must explore all choices.

*   **How it Works (Backtracking):**
    1.  **State Tracking:** The function maintains the current `path` (the order being built) and a snapshot of the `remaining_in_degree` for all nodes.
    2.  **Iteration:** At each step, it checks all nodes in the graph. If a node meets two criteria—it is not already in the current `path` AND its `remaining_in_degree` is 0—it is a valid choice for the next step.
    3.  **Exploration (Choice):** The algorithm selects such a valid node, adds it to the path, and temporarily updates the `remaining_in_degree` of all its neighbors (decrementing their prerequisites). It then makes a recursive call.
    4.  **Backtracking (Rollback):** Once the recursive call returns (meaning that particular path has been fully explored or led to a dead end), the algorithm *removes* the node from the path and restores the state of the degrees. This allows the loop to try the next available valid node.
    5.  **Goal:** When the `path` contains all nodes, a complete valid topological order has been found and is stored in `results`.

*   **Time Complexity:** This algorithm is significantly slower. In the worst case (a graph with few dependencies), the number of valid topological sorts can be factorial, making the complexity potentially exponential, $O(N!)$, where $N$ is the number of nodes.
