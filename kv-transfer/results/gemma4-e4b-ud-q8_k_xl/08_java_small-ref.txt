## 🧠 Program Functionality

The `LRUCache` class provides two main operations: `put` and `get`.

### 1. `get(K key)` (Accessing Data)
When you request a value by its key:
1. The cache first checks if the key exists in the underlying map.
2. If it exists, the corresponding node is retrieved.
3. Crucially, the node is then **moved to the front of the linked list** (making it the Most Recently Used, or MRU).
4. The value is returned.
5. If the key does not exist, `null` is returned.

### 2. `put(K key, V value)` (Inserting or Updating Data)
When you insert or update a key-value pair:
1. **Update:** If the key already exists, the value is updated, and the node is immediately moved to the front (MRU position).
2. **Eviction Check:** If the key is new, the cache checks if it is already at capacity (`map.size() >= capacity`).
3. **Evict LRU:** If the cache is full, the node at the **tail of the linked list** (the LRU item) is removed from both the linked list and the map.
4. **Insertion:** A new node is created, placed into the map, and added to the front of the linked list (MRU position).

The `printState()` method simply visualizes the current state of the cache, showing items ordered from MRU (left) to LRU (right).

## 🏗️ Data Structure Description

The LRU Cache is implemented by combining two core data structures: a **HashMap** and a **Doubly Linked List**. This combination allows for both fast lookups and fast order maintenance, giving the cache an optimal time complexity of $O(1)$ for both `get` and `put` operations.

### 1. The HashMap (`Map<K, Node<K, V>> map`)
*   **Role:** Provides instant (average $O(1)$) access to any node based on its key.
*   **Mechanism:** The map stores the `key` and maps it directly to a reference of the corresponding `Node` object in the linked list. This allows the cache to jump immediately to the location of an item without iterating through the list.

### 2. The Doubly Linked List (using `Node<K, V>`)
*   **Role:** Maintains the usage order of all items in the cache.
*   **Mechanism:** Each `Node` contains pointers to its `prev` (previous) and `next` nodes.
    *   **MRU:** The node closest to the `head` (the beginning of the list) is the Most Recently Used.
    *   **LRU:** The node closest to the `tail` (the end of the list) is the Least Recently Used.
*   **Sentinels:** The program uses two special, non-data-holding nodes, `head` and `tail`. These are called **sentinel nodes**. They act as fixed boundaries for the list, which greatly simplifies the logic for inserting, removing, and managing edge cases (e.g., an empty list or a list with only one item).

### Summary of Interaction

*   When an operation occurs, the **HashMap** finds the target `Node` instantly.
*   The **Doubly Linked List** uses the node reference to efficiently bypass and rearrange pointers (via `remove` and `addToFront`), ensuring the list order is always correct.
