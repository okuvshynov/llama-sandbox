## ⚙️ Code Breakdown (Function by Function)

### 1. `compress(input: &[u8]) -> Vec<(u8, u16)>`
This function performs the Run-Length Encoding compression.

*   **Input:** A byte slice (`&[u8]`), which is the original data.
*   **Output:** A vector of tuples (`Vec<(u8, u16)>`), where each tuple is `(the_byte, the_count_of_consecutive_bytes)`.
*   **Logic:**
    1. It initializes the process by assuming the first byte is the start of a run, setting `count` to 1.
    2. It iterates through the rest of the bytes.
    3. **If** the current byte matches the `current` byte of the run, it increments `count`.
    4. **If** the current byte does *not* match the run byte, or if the count would exceed the maximum capacity (`u16::MAX`), it means the current run has ended. It saves the finished run `(current, count)` to the `result`, and then starts a new run with the new byte.
    5. Crucially, after the loop finishes, the **last run** must be saved (since the `else` block isn't hit for it).

### 2. `decompress(data: &[(u8, u16)]) -> Vec<u8>`
This function reverses the RLE compression.

*   **Input:** A slice of compressed pairs (`&[(u8, u16)]`).
*   **Output:** The original byte slice (`Vec<u8>`).
*   **Logic:**
    1. It iterates through every `(byte, count)` pair in the input data.
    2. For each pair, it uses a nested loop that runs `count` times.
    3. In each iteration of the inner loop, it appends the `byte` to the `result` vector.

### 3. `analyze(input: &[u8]) -> HashMap<u8, usize>`
This utility function calculates the frequency of each unique byte in the original data.

*   **Input:** The original byte slice (`&[u8]`).
*   **Output:** A `HashMap<u8, usize>`, mapping each unique byte value to the number of times it appeared.
*   **Logic:**
    1. It initializes an empty `HashMap`.
    2. It iterates through every byte (`b`) in the input.
    3. `freq.entry(b).or_insert(0)`: This is a common Rust pattern. It checks if `b` is already a key in the map. If it is, it returns a mutable reference to its current value (the count). If it's not, it inserts `b` with a default value of `0` and returns a mutable reference to that new value.
    4. `*... += 1`: It then increments the count retrieved (or inserted).

### 4. `main()`
This function orchestrates the entire process.

*   **Setup:** It defines the input data: `b"AAABBBCCCCDDDDDEEEEE"`.
*   **Execution:**
    1. It calls `compress(data)` to get the compressed representation.

*   **Reporting:** It prints statistics about the operation (original size, compressed entries, compression ratio, and a roundtrip validation check).
*   **Frequency Output:** It takes the frequency map, converts it into a vector of pairs, sorts these pairs by count (descending), and prints the results.
