### 🐞 Bug 1: Memory Leak in `table_put` (Update Scenario)

**Location:** `table_put` function, inside the key-found loop.

**The Problem:**
When `table_put` finds an existing key, it updates the entry by calling `t->entries[i].value = strdup(value);`. The `strdup` function allocates new memory for the new `value` string. However, the original memory block previously pointed to by `t->entries[i].value` (which holds the old value) is never freed. This leads to a memory leak every time a value is updated for an existing key.

**Relevant Code:**
```c
// Inside table_put
if (strcmp(t->entries[i].key, key) == 0) {
    // BUG: The old value string allocated by a previous call to strdup() 
    // is lost and not freed before being overwritten.
    t->entries[i].value = strdup(value); 
    return;
}
```

**How to Fix It:**
Before overwriting the value pointer, the old value string must be freed.

```c
// Fix for Bug 1
if (strcmp(t->entries[i].key, key) == 0) {
    free(t->entries[i].value); // Free the old string
    t->entries[i].value = strdup(value); // Allocate and assign the new string
    return;
}
```
