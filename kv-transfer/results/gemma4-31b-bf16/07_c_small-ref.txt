### 1. Memory Leak on Value Update
In `table_put`, when a key already exists in the table, the code updates the value:
```c
if (strcmp(t->entries[i].key, key) == 0) {
    t->entries[i].value = strdup(value); // BUG: Memory Leak
    return;
}
```
**The Bug:** `t->entries[i].value` already points to a string allocated by a previous `strdup`. By assigning a new pointer to it without calling `free()` on the old one, the original string is leaked.
**The Fix:** Call `free(t->entries[i].value);` before assigning the new `strdup`.

### 2. Memory Leak on Entry Deletion
In `table_delete`, the program uses the "swap with last" method to remove an item in $O(1)$ time:
```c
if (strcmp(t->entries[i].key, key) == 0) {
    t->entries[i] = t->entries[t->size - 1]; // BUG: Memory Leak
    t->size--;
    return;
}
```
**The Bug:** The `Entry` struct contains pointers to heap-allocated memory (`key` and `value`). When you overwrite `t->entries[i]` with the last entry in the array, the pointers to the strings of the deleted entry are lost forever without being freed.
**The Fix:** Call `free()` on both `t->entries[i].key` and `t->entries[i].value` before performing the swap.

### 3. Unsafe `realloc` Usage
In `table_put`, the table is resized when it reaches capacity:
```c
t->entries = realloc(t->entries, sizeof(Entry) * t->capacity);
```
**The Bug:** If `realloc` fails, it returns `NULL`. By assigning the result directly back to `t->entries`, you overwrite the only pointer to the existing memory block. This results in a total memory leak of the entire entries array and makes it impossible to recover or shut down the program gracefully.
**The Fix:** Assign the result of `realloc` to a temporary pointer. If the temporary pointer is not `NULL`, assign it to `t->entries`.

### 4. Unchecked Allocations (Robustness)
Throughout the program, `malloc`, `realloc`, and `strdup` are called without checking if the return value is `NULL`. In a real-world system, if the system runs out of memory, the program will crash with a segmentation fault the moment it tries to dereference those pointers.
