This Python program implements **Topological Sorting** for a Directed Acyclic Graph (DAG). 

In simple terms, it takes a list of dependencies (where one task must be completed before another) and determines the valid order in which those tasks should be performed. In the provided example, the tasks represent a software build pipeline (e.g., you must "compile" before you can "link").

### The Algorithms

The program uses two different algorithmic approaches to solve this problem:

#### 1. Kahn's Algorithm (used in `solve`)
The `solve` function implements **Kahn's Algorithm**, which is an efficient, iterative way to find a single valid topological sort.

*   **The Logic:**
    1.  **In-Degree Calculation:** It first calculates the "in-degree" of every node (the number of incoming edges/dependencies).
    2.  **The Queue:** It identifies all nodes with an in-degree of $0$ (nodes that have no dependencies) and adds them to a queue.
    3.  **Processing:** It removes a node from the queue, adds it to the final result list, and "removes" its outgoing edges. 
    4.  **Updating Neighbors:** For every neighbor of the removed node, it decrements their in-degree. If a neighbor's in-degree drops to $0$, it means all its prerequisites are met, and it is added to the queue.
    5.  **Cycle Detection:** If the final result list contains fewer nodes than the total number of nodes in the graph, it means the graph contains a **cycle** (a circular dependency, like A $\to$ B $\to$ A), making a topological sort impossible.

**Complexity:** Time complexity is $O(V + E)$, where $V$ is the number of vertices and $E$ is the number of edges.

#### 2. Backtracking / DFS (used in `find_all`)
The `find_all` function uses a recursive **backtracking** approach to find *every* possible valid sequence.

*   **The Logic:**
    1.  **State Exploration:** Instead of using a simple queue, the function looks at the current state of the graph and finds **all** nodes that currently have an in-degree of $0$ and haven't been visited yet.
    2.  **Recursive Branching:** For every available node, it "picks" that node, updates the in-degrees of its neighbors, and recursively calls itself to find the next node in the sequence.
    3.  **Backtracking:** Once a path is completed or a dead end is reached, the function "undoes" the last step (pops the node and restores the in-degrees) to explore other possible branches.
    4.  **Collection:** Every time the path length equals the total number of nodes, a valid permutation has been found and is added to the results list.

**Complexity:** The time complexity is significantly higher (potentially exponential) because it explores all possible valid permutations of the tasks.
