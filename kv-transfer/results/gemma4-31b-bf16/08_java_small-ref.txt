This Java program implements an **LRU (Least Recently Used) Cache**. 

An LRU Cache is a specialized cache that stores a limited number of items. Once the cache reaches its maximum capacity, it removes the item that hasn't been accessed for the longest period of time to make room for new entries.

---

### How the Program Works

The program manages data using two primary operations:

1.  **`get(K key)`**: 
    *   It checks if the key exists in the cache.
    *   If it **doesn't exist**, it returns `null`.
    *   If it **does exist**, it marks the item as "recently used" by moving it to the front of the list and returns the value.

2.  **`put(K key, V value)`**:
    *   If the key **already exists**, it updates the value and moves the node to the front.
    *   If the key is **new**:
        *   It checks if the cache is at full capacity. If so, it deletes the "Least Recently Used" item (the one at the back of the list) from both the map and the list.
        *   It then inserts the new item at the front of the list and adds it to the map.

---

### The Data Structure: A Hybrid Approach

To achieve high performance, this program uses a combination of two data structures: a **HashMap** and a **Doubly Linked List**. This is a classic design pattern used to ensure that both adding and retrieving data happen in **constant time—$O(1)$**.

#### 1. The HashMap (`Map<K, Node<K, V>>`)
The HashMap acts as an index. It maps keys directly to their corresponding nodes in the linked list. 
*   **Purpose:** Without the map, finding a specific item in a linked list would take $O(n)$ time (you would have to loop through the list). The map allows the program to jump directly to the node in $O(1)$ time.

#### 2. The Doubly Linked List (`Node<K, V>`)
The doubly linked list maintains the "age" or "recency" of the items. 
*   **The Front (Head):** Represents the **Most Recently Used (MRU)** item.
*   **The Back (Tail):** Represents the **Least Recently Used (LRU)** item.
*   **Purpose:** A doubly linked list allows the program to remove a node from the middle and move it to the front in $O(1)$ time, provided you already have a reference to that node (which the HashMap provides).

#### 3. Sentinel Nodes (Dummy Head and Tail)
You will notice the constructor creates two empty nodes: `head` and `tail`.
*   These are called **sentinel nodes**. They do not hold actual data; they simply act as boundary markers.
*   **Purpose:** They simplify the code by eliminating the need to check for `null` whenever the program adds or removes nodes from the ends of the list.
