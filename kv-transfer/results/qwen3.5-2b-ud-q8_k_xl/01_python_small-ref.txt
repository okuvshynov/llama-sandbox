```python
from collections import deque

def running_median(stream):
    window = deque()
    medians = []
    for i, val in enumerate(stream):
        window.append(val)
        if len(window) > 5:
            window.popleft()
        medians.append(find_median(window))
    return medians
```

### Why This Fixes It

- `deque` offers O(1) time complexity for both appending and removing elements from the front.
- This ensures that the sliding window operation remains efficient even as the stream grows.
- The `find_median` function remains unaffected, as it only depends on the current window contents.
