## Task Scheduler (scheduler.go)

### Race Conditions
1. **Task registration in SubmitTask**: The task is added to both `s.tasks` map and `heap` queue simultaneously. While mutex protects the map, `heap.Push` might have race issues with the heap operations.

2. **Task state transitions**: When a task completes, the dependency resolution happens in `CompleteTask`, but there's a race between updating task state and adding to heap.

3. **Worker heartbeat race**: Multiple workers can receive the same heartbeat and mark themselves as dead simultaneously.

4. **Task cancellation race**: Task cancellation happens in `CancelTask`, but there's a race between updating task state and checking if task exists in `s.completed`.

5. **Dependency resolution race**: Dependencies are resolved in `CompleteTask`, but there's a race with the heap operations.

### Performance Issues
1. **Queue size limit**: The queue is limited to `maxQueueSize`, but there's no eviction policy for failed tasks. Tasks should be requeued with exponential backoff, not just rejected.

2. **Heartbeat interval**: Heartbeat checks every `workerTimeout / 2`, but there's no timeout for workers that stay alive.

3. **Dependency resolution**: Full dependency resolution happens on every task completion, which is O(n²) in worst case.

4. **Resource contention**: Multiple goroutines updating the same map and heap simultaneously.

5. **Heap operations**: `heap.Push` and `heap.Pop` are unbuffered, causing potential contention.

### Design Problems
1. **Task ID generation**: Uses nanosecond timestamp which is not unique across different servers.

2. **No task priority queue**: Task priority is stored but not used for queuing.

3. **No task cancellation support**: Tasks can be cancelled, but there's no mechanism to cancel all tasks for a worker.

4. **No task timeout enforcement**: Tasks can run indefinitely (though `Timeout` field exists).

5. **No graceful task cancellation**: Tasks are not properly cancelled when they exceed timeout.

6. **Worker pool doesn't support task migration**: Workers can't be reassigned tasks.

7. **No task retry with exponential backoff**: Tasks retry but don't have a proper retry policy.

8. **No task isolation**: Tasks share the same heap and queue.

9. **No task priority enforcement**: Priority is stored but not used for scheduling.
