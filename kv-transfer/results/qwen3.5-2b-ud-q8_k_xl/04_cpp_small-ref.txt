This C++ program implements **Huffman coding**, a lossless data compression technique that assigns shorter bit sequences to more frequent characters in a text. It uses a priority queue to construct the Huffman tree, generates codes based on character frequencies, and encodes the input string into a binary format.

---

### What the Program Does

1. **Builds a Huffman Tree**:
   - Counts character frequencies from input text.
   - Creates a priority queue of nodes ordered by frequency.
   - Merges nodes with the two lowest frequencies to build the tree.

2. **Generates Huffman Codes**:
   - Each node in the tree becomes a character with a unique binary code (prefix).
   - Empty and leaf nodes are assigned codes using depth-based logic.

3. **Encodes Text**:
   - Applies the generated codes to each character in the input string.

4. **Displays and Measures Compression**:
   - Prints the generated codes.
   - Calculates and reports the original vs. encoded size and compression ratio.

---

### Time Complexity Analysis

| Operation                        | Complexity  | Notes |
|----------------------------------|-------------|-------|
| Frequency Counting (text)        | O(n)        | Scans input string once |
| Priority Queue Initialization    | O(n)        | Adds all unique characters |
| Tree Construction (Merges)       | O(n log n)  | Each of n-1 merges takes O(log n) |
| Code Generation (buildCodes)     | O(n)        | Visits each node once |
| Sorting Codes                    | O(m log m)  | m ≤ n, where m is unique characters |
| Encoding                         | O(n)        | Concatenates codes for each character |
| **Overall Time Complexity**      | **O(n log n)** | Dominated by priority queue merges |

> **n** = length of input text  
> **m** = number of unique characters

---

### Space Complexity Analysis

| Resource     | Complexity | Notes |
|--------------|------------|-------|
| Node Memory  | O(n)       | Each character creates a Node |
| Priority Queue | O(n)     | Stores all nodes |
| Codes Map    | O(n)       | Stores all unique characters and codes |
| Output String | O(n)       | Encoded text |
| **Total Space** | **O(n)** | Linear in input size |

---

### Key Implementation Details

- **Priority Queue (`std::priority_queue`)**: Implements a max-heap by default. Custom comparator `Compare` ensures higher frequencies are processed first.
- **Node Structure**: Stores character, frequency, and pointers to left/right children.
- **Huffman Tree Construction**: Uses a priority queue to merge two nodes with lowest frequencies until one root remains.
- **Code Generation**: Uses depth-first traversal to assign binary strings (0 or 1) for each character.
- **Compression Ratio Calculation**: Compares input size in bits (`text.size() * 8`) with encoded size (`encoded.size()`).
