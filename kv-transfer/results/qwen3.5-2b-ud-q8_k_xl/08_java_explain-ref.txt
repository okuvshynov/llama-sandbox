This Java program implements a **Least Recently Used (LRU) Cache** using a **doubly linked list** for tracking access order, with a **HashMap** for efficient key-value storage.

### Program Overview

The `LRUCache` class maintains a limited-capacity cache where the least recently accessed items are automatically removed when the cache is full. It supports `get` and `put` operations that are both efficient and follow the LRU eviction policy.

### Data Structure Breakdown

1. **Node<K, V> (Internal Node)**:
   - Represents a single cache entry.
   - Maintains a `key`, `value`, and references to `prev` and `next` nodes.
   - This forms a **doubly linked list** that tracks access order.

2. **Map<K, Node<K, V>> map**:
   - A HashMap that stores key → Node mappings for O(1) access.
   - Enables efficient retrieval and updates of existing or new entries.

3. **head and tail**:
   - `head` points to the first node in the doubly linked list.
   - `tail` points to the last node.
   - These sentinel nodes are used to manage list boundaries and insertion at the front.

4. **Capacity**:
   - The maximum size the cache can hold.
   - Exceeding this triggers eviction of the least recently used item.

### LRU Behavior in Practice

- **get(key)**: Moves the accessed key to the front of the list (most recently used).
- **put(key, value)**:
  - If the key exists, updates its value and moves it to the front.
  - If the key does not exist and the cache is full, removes the least recently used item (tracked at `tail.prev`) and adds the new key-value pair.
