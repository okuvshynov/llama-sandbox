## Algorithm Description

### Topological Sort (solve function)
This is a classic topological sorting algorithm:
1. Find all nodes with in-degree 0 (no incoming edges)
2. Add them to a queue
3. Process each node in queue:
   - Mark it as visited
   - For each neighbor, decrease its in-degree
   - If a neighbor's in-degree becomes 0, add to queue
4. Return the processed nodes

### Backtracking (find_all function)
This finds all valid topological permutations using DFS:
1. Start with empty path and full in-degree map
2. Try adding each node with remaining in-degree 0
3. Recursively explore valid paths
4. Backtrack when exploring

- Graph with cycles will return None (not all nodes visited)
- Multiple valid orderings are captured
- Unreachable nodes are handled (have in-degree 0)

### Program Overview

### Algorithm Explanations

#### 1. `build_graph(edges)`
- Initializes a graph using a `defaultdict(list)` for adjacency.
- Tracks in-degree of each node to detect dependencies.
- Returns the graph, in-degree map, and set of all nodes.

#### 2. `solve(edges)` (Kahn's Algorithm)
- Uses a queue of nodes with in-degree 0.
- Iteratively removes processed nodes, decreasing neighbors' in-degree.
- When a neighbor's in-degree becomes 0, it is added to the queue.
- Returns the resulting topological ordering if all nodes are processed.

#### 3. `find_all(edges)` (Backtracking DFS)
- Uses recursive DFS to explore all valid paths.
- Builds a path node-by-node, ensuring no node appears after its dependencies.
- Backtracks when a node is no longer valid for the current path.
- Collects all valid topological orderings.
