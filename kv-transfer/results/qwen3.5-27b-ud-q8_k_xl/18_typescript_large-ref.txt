### 1. Security Vulnerabilities (Critical)

#### A. SQL Injection Vulnerabilities
**Issue:** Several queries use string interpolation instead of parameterized queries, allowing attackers to manipulate database commands.
*   **Location:** `UserRepository.findByEmail`, `AuditLogRepository.findByUser`, `DocumentRepository.search`, `SearchService.getSuggestions`.
*   **Example:**
    ```typescript
    // UserRepository.findByEmail
    "SELECT * FROM users WHERE email = '" + email + "'"
    ```
    **Impact:** An attacker can log in as `admin' --` and bypass authentication or dump the database.
*   **Fix:** Use parameterized queries (`$1`, `$2`) consistently.
    ```typescript
    // Fixed
    "SELECT * FROM users WHERE email = $1", [email]
    ```

#### B. Broken Password Authentication
**Issue:** The password verification logic in `AuthService.login` does not match the hashing logic in `register`.
*   **Location:** `AuthService.login` vs `AuthService.register`.
*   **Logic:**
    1.  **Register:** `hash = sha256(password + salt)`
    2.  **Login:** `hash = sha256(password)` (Salt is ignored during verification).
