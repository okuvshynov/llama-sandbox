This C++ program implements the Huffman coding algorithm, a method for lossless data compression that assigns variable-length binary codes to characters based on their frequencies.

---

### Program Overview

#### 1. Data Structures
- **`Node` struct**: Represents a node in the Huffman tree with a character, its frequency, and pointers to left and right children.
- **`Compare` struct**: Used to create a min-heap by comparing node frequencies.

#### 2. Key Functions

- **`buildTree()`**
  - Counts character frequencies in the input text.
  - Builds a min-heap using a priority queue.
  - Repeatedly merges the two lowest-frequency nodes until one root node remains.
  - Returns the root of the Huffman tree.

- **`buildCodes()`**
  - Recursively traverses the Huffman tree to assign binary codes (0 for left, 1 for right).
  - Stores the resulting codes in a hash map for quick lookup.

- **`encode()`**
  - Converts the original text into a bit string using the generated Huffman codes.

- **`main()`**
  - Initializes a sample text.
  - Builds the Huffman tree and generates codes.
  - Sorts and prints the codes by length.
  - Displays compression statistics: original size, encoded size, and compression ratio.

---

### Time Complexity Analysis

| Step                        | Complexity        | Explanation |
|----------------------------|-------------------|-------------|
| Frequency Counting         | O(n)              | Single pass over input text |
| Heap Construction          | O(k log k)        | k = number of unique characters |
| Huffman Tree Construction  | O(k log k)        | k–1 merge operations |
| Code Generation            | O(k × m)          | m = average code length |
| Encoding Text              | O(n × m)          | Each character replaced by its code |
| Sorting Codes              | O(k log k)        | Sorting unique characters by code length |

**Overall Time Complexity:**  
**O(n × m + k log k)**  
- n = length of input text  
- k = number of unique characters (≤ 256 for ASCII)  
- m = average code length (typically 5–15 bits)

Since k is small and m is bounded, the dominant term is usually **O(n × m)**, or simply **O(n)** in practical scenarios.

---

### Space Complexity Analysis

| Component                    | Space Usage     |
|-----------------------------|-----------------|
| Frequency Map               | O(k)            |
| Priority Queue              | O(k)            |
| Huffman Tree                | O(k)            |
| Code Map                    | O(k × m)        |
| Encoded Output              | O(n × m)        |

**Overall Space Complexity:**  
**O(n × m + k × m)**  
In practice, this simplifies to **O(n)** for large input texts.
