### Program Overview
This Java program implements an **LRU (Least Recently Used) Cache**.

An LRU Cache is a data structure that stores a fixed number of items (the `capacity`). When the cache is full and a new item needs to be added, the system automatically removes the item that has not been accessed for the longest time. This is commonly used in web browsers, operating systems, and databases to speed up access to frequently used data.

The program defines a generic class `LRUCache<K, V>` where `K` is the key type and `V` is the value type. It provides two main operations:
1.  **`get(key)`**: Retrieves the value associated with a key. If the key exists, it marks it as "recently used."
2.  **`put(key, value)`**: Inserts or updates a key-value pair. If the cache is full, it evicts the least recently used item before adding the new one.

### Data Structure Description
The program uses a **hybrid data structure** combining two fundamental structures to achieve **$O(1)$ time complexity** for both `get` and `put` operations:

1.  **Hash Map (`HashMap<K, Node<K, V>>`)**
    *   **Purpose:** Provides fast lookup.
    *   **Function:** Maps a Key directly to its corresponding `Node` object.
    *   **Benefit:** Allows the program to find a specific node in $O(1)$ time without traversing the list.

2.  **Doubly Linked List (`Node`)**
    *   **Purpose:** Maintains the order of usage.
    *   **Function:** The `Node` class contains `prev` and `next` pointers.
    *   **Ordering:**
        *   **Head (Front):** Represents the **Most Recently Used (MRU)** item.
        *   **Tail (Back):** Represents the **Least Recently Used (LRU)** item.
    *   **Sentinel Nodes:** The `head` and `tail` variables are dummy nodes (initialized with `null` values). They simplify edge cases (like an empty list or removing the only node) so the `remove` and `addToFront` logic doesn't need to check for `null` pointers constantly.
