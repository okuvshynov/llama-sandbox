This Rust program demonstrates a simple **Run-Length Encoding (RLE)** compression algorithm. It compresses a byte sequence, decompresses it back to verify correctness, analyzes the frequency of characters, and prints statistics about the compression efficiency.

Here is the step-by-step breakdown of the code:

### 1. Function: `compress`
**Purpose:** Converts a sequence of bytes into a list of `(byte, count)` pairs.
**Logic:**
1.  **Initialization:** Creates an empty result vector. If the input is empty, it returns immediately.
2.  **Tracking:** Sets the `current` byte to the first byte of the input and `count` to 1.
3.  **Iteration:** Loops through the rest of the input bytes.
    *   **Match:** If the next byte equals `current` and the count hasn't hit the maximum (`u16::MAX`), it increments `count`.
    *   **Mismatch:** If the byte changes (or count is full), it pushes `(current, count)` to the result, resets `current` to the new byte, and resets `count` to 1.
4.  **Final Push:** After the loop finishes, it pushes the final accumulated `(current, count)` pair.
5.  **Return:** Returns the vector of tuples.

### 2. Function: `decompress`
**Purpose:** Reverses the compression process.
**Logic:**
1.  **Initialization:** Creates an empty result vector.
2.  **Iteration:** Loops through every `(byte, count)` tuple in the compressed data.
3.  **Expansion:** For each tuple, it runs an inner loop `count` times, pushing the `byte` into the result vector each time.
4.  **Return:** Returns the fully reconstructed byte vector.

### 3. Function: `analyze`
**Purpose:** Calculates how many times each unique byte appears in the input.
