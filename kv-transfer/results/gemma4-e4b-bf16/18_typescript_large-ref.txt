## 🔴 Security Vulnerabilities (CRITICAL)

Security is the most immediate and severe concern in this codebase.

### 1. SQL Injection Risks (Critical)
The repository layer frequently constructs SQL queries using string concatenation, making the system highly vulnerable to SQL Injection.

**Affected Areas:**
*   `UserRepository.findByEmail`:
    ```typescript
    // VULNERABLE:
    "SELECT * FROM users WHERE email = '" + email + "'"
    ```
*   `DocumentRepository.search`:
    ```typescript
    // VULNERABLE:
    whereClause += ` AND (title ILIKE '%${query}%' OR content ILIKE '%${query}%')`;
    ```
    *The reliance on positional parameters (`$1`, `$2`) is undermined by mixing them with raw string concatenation in the `WHERE` clause.*
*   `AuditLogRepository.findByUser`:
    ```typescript
    // VULNERABLE:
    `SELECT * FROM audit_log WHERE user_id = $1 AND timestamp BETWEEN '${startDate}' AND '${endDate}' ORDER BY timestamp DESC`,
    ```

**Suggested Fixes:**
1.  **Strictly use parameterized queries** for *all* user-provided inputs, even when constructing clauses.
2.  For `ILIKE` queries, pass the wildcards (`%`) as part of the parameter value:
    ```typescript
    // SAFE FIX for search:
    whereClause += ` AND (title ILIKE $${paramIdx} OR content ILIKE $${paramIdx})`;
    params.push(`%${query}%`);
    paramIdx++;
    ```
3.  For date range queries, always pass dates as parameters, not interpolated strings:
    ```typescript
    // SAFE FIX for audit log:
    await pool.query(
        `SELECT * FROM audit_log WHERE user_id = $1 AND timestamp BETWEEN $2 AND $3 ORDER BY timestamp DESC`,
        [userId, startDate, endDate]
    );
    ```

### 2. Weak Password Hashing (Critical)
The `AuthService` uses a simple SHA256 hash combined with a random salt, which is **not sufficient** for password storage. This is susceptible to brute-force and rainbow table attacks.

**Affected Area:**
*   `AuthService.register` and `AuthService.login`

**Suggested Fix:**
Use a modern, slow, and adaptive hashing library like **`bcrypt`** or **`argon2`**. These algorithms are designed to be computationally expensive, slowing down attackers.
