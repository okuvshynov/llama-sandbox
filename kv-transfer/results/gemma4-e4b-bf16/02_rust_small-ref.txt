## 🧩 Function Breakdown

### 1. `fn compress(input: &[u8]) -> Vec<(u8, u16)>`
This function performs Run-Length Encoding (RLE). It reads a sequence of bytes and replaces consecutive runs of the same byte with a pair: `(byte, count)`.

*   **Input:** A byte slice (`&[u8]`), which is the original data (e.g., `b"AAABBB"`).
*   **Output:** A vector of tuples (`Vec<(u8, u16)>`), where each tuple is `(the byte value, the number of times it appeared consecutively)`.
*   **Logic:**
    1.  It initializes the result vector and handles the edge case where the input is empty.
    2.  It starts by assuming the first byte is the `current` character and the `count` is 1.
    3.  It iterates through the rest of the input bytes (`input[1..]`).
    4.  **If** the current byte matches the `current` character, the `count` is incremented (provided it doesn't exceed `u16::MAX`).
    5.  **If** the current byte is different, or if the count limit is reached, the program records the accumulated run (`(current, count)`) into the `result`. It then resets the process: the new byte becomes the new `current` character, and the `count` resets to 1.
    6.  Finally, after the loop finishes, it must push the last run onto the result vector, as the loop logic only pushes a run when a *change* is detected.

*   **Example:** `b"AAAB"` becomes `[(A, 3), (B, 1)]`.

### 2. `fn decompress(data: &[(u8, u16)]) -> Vec<u8>`
This function reverses the RLE process. It expands the compressed data back into its original sequence.

*   **Input:** A slice of compressed tuples (`&[(u8, u16)]`).
*   **Output:** A vector of bytes (`Vec<u8>`), the original data.
*   **Logic:**
    1.  It initializes an empty result vector.
    2.  It iterates through every `(byte, count)` pair in the compressed data.
    3.  For each pair, it uses a nested loop (`for _ in 0..count`) to push the specified `byte` into the result vector exactly `count` times.

*   **Example:** `[(A, 3), (B, 1)]` becomes `[A, A, A, B]`.

### 3. `fn analyze(input: &[u8]) -> HashMap<u8, usize>`
This utility function calculates the frequency of every unique byte in the input data.

*   **Input:** A byte slice (`&[u8]`).
*   **Output:** A `HashMap<u8, usize>`, mapping each unique byte value (key) to how many times it appeared (value).
*   **Logic:**
    1.  It initializes an empty `HashMap`.
    2.  It iterates through every byte in the input.
    3.  For each byte (`b`), it uses the `entry().or_insert(0)` pattern to efficiently check if the byte is already in the map. If it is, it gets the existing count; if not, it inserts `0` and gets a mutable reference to that new zero.
    4.  It then increments the count for that byte.
