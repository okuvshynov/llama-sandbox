## ⚙️ Algorithm Descriptions

### 1. `solve(edges)`: Topological Sort (Kahn's Algorithm)

**Goal:** To find a single valid sequence in $O(V+E)$ time, where $V$ is the number of vertices (tasks) and $E$ is the number of edges (dependencies).

**How it works:**

1.  **Initialization:** The graph and in-degrees are constructed. A queue is initialized with all nodes that have an in-degree of 0 (tasks with no prerequisites, meaning they can start immediately).
2.  **Processing:** While the queue is not empty:
    a. A node ($u$) is dequeued and added to the result list.
    b. For every neighbor ($v$) of $u$ (i.e., every task that depended on $u$):
        i. The dependency count of $v$ is reduced by 1 (`in_degree[v] -= 1`).
        ii. If $v$'s in-degree drops to 0, it means all of its prerequisites are now met, and $v$ is added to the queue.
3.  **Cycle Detection:** If the final `result` list does not contain all the original nodes, it means the graph contains a cycle (a circular dependency, e.g., Task A depends on B, and B depends on A). In this case, no valid order exists, and the function returns `None`.

### 2. `find_all(edges)`: Backtracking Search

**Goal:** To find every single valid permutation of the nodes that satisfies the dependency constraints. This process is computationally expensive (it can be exponential) because it must explore all branching possibilities.

**How it works:**

This function uses a recursive technique called **Backtracking**. The function maintains the current state of the partial ordering (`path`) and the remaining dependencies (`remaining_in_degree`).

1.  **Base Case:** If the length of the current `path` equals the total number of nodes, a complete, valid ordering has been found, and it is saved.
2.  **Recursive Step:** The algorithm iterates through every node in the entire set of tasks. For each node, it checks two conditions:
    a. **Is it unused?** The node must not already be in the current `path`.
    b. **Are its dependencies met?** Its current `remaining_in_degree` must be 0.
3.  **Selection & Transition:** If a node meets both criteria, it is a valid next step:
    a. The node is added to the current `path`.
    b. **Crucial Step (State Management):** A *copy* of the dependency state (`remaining_in_degree`) is created. For every neighbor of the selected node, its dependency count is decremented in this *copy*. This simulates the completion of the current task.
    c. The function calls itself recursively (`backtrack`) with the updated path and the modified dependency state.
    d. **Backtrack:** After the recursive call returns, the node is removed from the current `path` (`path.pop()`). This undoes the choice, allowing the algorithm to explore other potential tasks that could have been chosen at this step.
