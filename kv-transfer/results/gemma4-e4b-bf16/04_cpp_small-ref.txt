## 📖 Program Explanation

The program executes four main stages:

### 1. Frequency Analysis and Tree Construction (`buildTree`)
*   **Frequency Counting:** It first iterates through the input string (`"this is an example..."`) and counts how many times each unique character appears.
*   **Priority Queue Setup:** It uses a **Min-Heap** (implemented via `std::priority_queue` with a custom `Compare` struct) to store nodes, ordered by their frequency. Initially, each unique character becomes a leaf node in this heap.
*   **Tree Building:** The core Huffman process begins in a loop:
    1.  It extracts the two nodes with the lowest frequencies (the two "cheapest" nodes).
    2.  It creates a new internal parent node whose frequency is the sum of the two extracted nodes.
    3.  The two extracted nodes become the left and right children of this new parent.
    4.  This new parent node is inserted back into the priority queue.
*   This process continues until only one node remains in the priority queue—this is the root of the complete Huffman Tree.

### 2. Code Generation (`buildCodes`)
*   The program performs a Depth First Search (DFS) traversal starting from the root of the Huffman Tree.
*   It maintains a `prefix` string, which represents the path taken from the root.
    *   Moving to the **left** child appends a `'0'` to the prefix.
    *   Moving to the **right** child appends a `'1'` to the prefix.
*   When the traversal reaches a leaf node (a node representing a character), the accumulated `prefix` is stored as the Huffman code for that character.

### 3. Output and Sorting
*   The program collects all the generated codes into a map.
*   It then converts this map into a vector of pairs and **sorts these pairs based on the length of the code**. This section demonstrates a helpful feature but is not strictly required for the compression itself.
*   Finally, it prints a list of all characters and their assigned binary codes.

### 4. Encoding and Comparison (`encode`)
*   The `encode` function iterates through the original input text character by character. For each character, it looks up its corresponding binary code in the generated map and concatenates that code to build the final compressed binary string (`encoded`).
*   The `main` function compares the size of the original data (calculated as $N \times 8$ bits, assuming standard 8-bit ASCII) to the size of the encoded data (the total number of bits in the Huffman codes) and calculates the compression ratio.
