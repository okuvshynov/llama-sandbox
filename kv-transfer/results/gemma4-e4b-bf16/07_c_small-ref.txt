## 🐞 Memory Bug 1: Memory Leak on Key Update (in `table_put`)

**Location:** `table_put` function, inside the loop where a matching key is found.

**The Bug:** When `table_put` finds an existing key and updates its value, it allocates new memory for the new value using `strdup(value)`. However, it never frees the memory previously held by `t->entries[i].value`. This leads to a memory leak every time an entry is updated.

**Code Snippet (Leaking code):**
```c
// In table_put
if (strcmp(t->entries[i].key, key) == 0) {
    // BUG: The previous value pointer is overwritten without being freed.
    t->entries[i].value = strdup(value); 
    return;
}
```

**How to Fix:** Before assigning the new value, the existing value string must be freed.

```c
// Fixed version (assuming only the value is updated):
if (strcmp(t->entries[i].key, key) == 0) {
    // Free the old memory first
    free(t->entries[i].value); 
    // Allocate new memory
    t->entries[i].value = strdup(value);
    return;
}
```
*(Note: If you wanted to allow updating the key as well, you would need similar logic to free the old key memory.)*
