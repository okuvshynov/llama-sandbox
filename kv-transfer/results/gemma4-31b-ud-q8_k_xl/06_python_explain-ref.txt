This Python program implements **Topological Sorting** on a Directed Acyclic Graph (DAG). 

In simple terms, it takes a list of dependencies (where one task must be completed before another) and determines the valid sequences in which those tasks can be performed. In the provided example, it is simulating a software build pipeline (e.g., you must resolve `deps` before you can `compile`).

### Detailed Algorithmic Explanation

#### 1. Graph Representation (`build_graph`)
Before sorting, the program converts the list of edge tuples into two data structures:
*   **Adjacency List (`graph`):** A dictionary where each key is a node and its value is a list of nodes that depend on it.
*   **In-Degree Map (`in_degree`):** A dictionary that counts how many incoming edges each node has. If a node has an in-degree of $0$, it means nothing needs to happen before it.

#### 2. Finding One Valid Order (`solve` function)
This function implements **Kahn's Algorithm**, which is an efficient iterative approach to topological sorting.

**The Process:**
1.  **Initialization:** Find all nodes with an `in_degree` of $0$ and put them into a queue.
2.  **Processing:** While the queue is not empty:
    *   Remove a node from the queue and add it to the `result` list.
    *   For every neighbor (node that depends on the current node), decrement its `in_degree` by $1$.
    *   If a neighbor's `in_degree` drops to $0$, add it to the queue.
3.  **Cycle Detection:** If the final `result` list contains fewer nodes than the total number of nodes in the graph, the program returns `None`. This indicates a **circular dependency** (a cycle), making a topological sort impossible.

**Time Complexity:** $O(V + E)$, where $V$ is the number of vertices (nodes) and $E$ is the number of edges.

#### 3. Finding All Valid Orders (`find_all` function)
Since there can be multiple valid sequences (e.g., `link` and `test` might be independent of each other), this function uses **Backtracking (Recursive DFS)**.

**The Process:**
1.  **Search:** It looks for all nodes that currently have an `in_degree` of $0$ and have not yet been added to the path.
2.  **Recurse:** For each available node, it:
    *   Adds the node to the current path.
    *   Temporarily reduces the `in_degree` of its neighbors.
    *   Calls itself recursively to find the next node in the sequence.
3.  **Backtrack:** After the recursive call returns, it removes the node from the path and restores the `in_degree` counts to explore other possible permutations.

**Time Complexity:** In the worst case (a graph with no edges), this is $O(V!)$, as it explores every possible permutation of nodes.
