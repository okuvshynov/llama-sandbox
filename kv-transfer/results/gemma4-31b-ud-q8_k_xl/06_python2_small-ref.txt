This Python program implements **Topological Sorting** on a Directed Acyclic Graph (DAG). 

In simple terms, it takes a list of dependencies (where one task must be completed before another) and determines the valid sequences in which those tasks can be performed. In the provided example, it is simulating a software build pipeline (e.g., you must "compile" before you can "link").

### Detailed Component Analysis

#### 1. `build_graph(edges)`
This is a helper function that converts the raw list of tuples into a usable graph structure:
*   **Adjacency List (`graph`)**: A dictionary where each key is a task and the value is a list of tasks that depend on it.
*   **In-Degree Map (`in_degree`)**: A dictionary that tracks how many prerequisites each task has. If a task has an in-degree of `0`, it can be started immediately.

#### 2. `solve(edges)` — Kahn's Algorithm
This function finds **one** valid topological order using **Kahn's Algorithm**. The logic follows these steps:
1.  **Initialize**: Find all nodes with an in-degree of `0` and put them into a queue.
2.  **Process**: While the queue is not empty:
    *   Pop a node (task) from the queue and add it to the final result list.
    *   For every neighbor (task that depends on the current one), decrement its in-degree by 1.
    *   If a neighbor's in-degree drops to `0`, add it to the queue.
3.  **Cycle Detection**: If the final result list contains fewer nodes than the total number of nodes in the graph, it means there is a **circular dependency** (a cycle), and no valid ordering exists.

#### 3. `find_all(edges)` — Backtracking
While `solve` finds any single valid path, `find_all` uses **recursion and backtracking** to find **every possible** valid sequence.
1.  It identifies all nodes currently eligible to be picked (in-degree is `0` and not yet in the path).
2.  It picks one eligible node, updates the dependencies of its neighbors, and recurses to find the next node.
3.  After the recursive call returns, it "backtracks" (removes the node from the path and restores the in-degrees) to try a different eligible node.

### The Algorithm: Topological Sort

**Topological Sorting** is an ordering of vertices in a directed graph such that for every directed edge $u \to v$, vertex $u$ comes before $v$ in the ordering.

#### Complexity Analysis:
*   **Time Complexity (`solve`)**: $O(V + E)$, where $V$ is the number of vertices (tasks) and $E$ is the number of edges (dependencies). Each node and edge is processed exactly once.
*   **Space Complexity**: $O(V + E)$ to store the graph and the in-degree map.
*   **Time Complexity (`find_all`)**: In the worst case (a graph with no edges), the complexity is $O(V!)$, as it explores every possible permutation of nodes.
