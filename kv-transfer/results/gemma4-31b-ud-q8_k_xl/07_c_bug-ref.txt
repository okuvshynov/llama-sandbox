Here is the detailed breakdown of the bugs:

### 1. Memory Leak in `table_put` (Updating Keys)
When you update the value of a key that already exists in the table, the program allocates new memory for the value but forgets to free the old one.
```c
if (strcmp(t->entries[i].key, key) == 0) {
    t->entries[i].value = strdup(value); // BUG: Old t->entries[i].value is leaked
    return;
}
```
**Fix:** Call `free(t->entries[i].value);` before assigning the new `strdup` result.

### 2. Memory Leak in `table_delete`
The `table_delete` function uses a "swap-with-last" strategy to keep the array contiguous. While efficient, it overwrites the pointers of the entry being deleted without freeing the strings they point to.
```c
if (strcmp(t->entries[i].key, key) == 0) {
    t->entries[i] = t->entries[t->size - 1]; // BUG: Key and Value of entry [i] are leaked
    t->size--;
    return;
}
```
**Fix:** You must `free(t->entries[i].key)` and `free(t->entries[i].value)` before performing the swap.

### 3. Unsafe `realloc` Usage
The program assigns the result of `realloc` directly back to the original pointer.
```c
t->entries = realloc(t->entries, sizeof(Entry) * t->capacity);
```
If `realloc` fails, it returns `NULL`. By assigning `NULL` directly to `t->entries`, you lose the pointer to the existing memory block, making it impossible to free that memory (a leak) and causing a crash when the program next tries to access the table.

**Fix:** Use a temporary pointer:
```c
Entry* temp = realloc(t->entries, sizeof(Entry) * t->capacity);
if (temp == NULL) { /* Handle error */ }
t->entries = temp;
```

### 4. Missing NULL Checks (Allocation Failures)
The program assumes that `malloc`, `realloc`, and `strdup` always succeed. In a production environment, if the system runs out of memory, these functions return `NULL`. Dereferencing these pointers immediately after allocation will cause a **Segmentation Fault**.
*   `table_create`: `malloc` for `t` and `t->entries` are unchecked.
*   `table_put`: `strdup` for `key` and `value` are unchecked.
*   `table_put`: `realloc` is unchecked.
