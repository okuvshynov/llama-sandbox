This program implements **Huffman Coding**, a famous algorithm used for **lossless data compression**. 

The goal of Huffman Coding is to reduce the amount of space needed to store a string of text by assigning shorter binary codes to characters that appear frequently and longer binary codes to characters that appear rarely.

#### **Step-by-Step Walkthrough:**
1.  **Frequency Counting:** It first counts how many times each character appears in the input string using a map (`unordered_map`).
2.  **Building the Huffman Tree:** 
    *   It places every unique character into a **Min-Priority Queue** (a heap), where the character with the lowest frequency is always at the top.
    *   It repeatedly removes the two nodes with the lowest frequencies, creates a new "parent" node with a frequency equal to the sum of the two, and pushes this parent back into the queue.
    *   This process continues until only one node remains: the **Root** of the Huffman Tree.
3.  **Generating Binary Codes:** 
    *   The program traverses the tree from the root to the leaves. 
    *   Moving **left** adds a `"0"` to the code; moving **right** adds a `"1"`.
    *   Characters located at the leaves of the tree are assigned these resulting binary strings.
4.  **Encoding:** It replaces every character in the original text with its newly assigned binary code.
5.  **Analysis:** Finally, it calculates the compression ratio by comparing the size of the original text (assuming 8 bits per character) against the size of the encoded binary string.

### **Time Complexity Analysis**

Let $n$ be the total number of characters in the input text and $k$ be the number of **unique** characters (the alphabet size).

| Step | Operation | Time Complexity | Reason |
| :--- | :--- | :--- | :--- |
| 1 | Frequency Map | $O(n)$ | We iterate through the entire string once. |
| 2 | Initializing Priority Queue | $O(k \log k)$ | We insert $k$ unique characters into the heap. |
| 3 | Building the Tree | $O(k \log k)$ | The loop runs $k-1$ times. Each `pop` and `push` operation in a priority queue takes $O(\log k)$. |

| 5 | Encoding the Text | $O(n)$ | We iterate through the original string and append the corresponding code. |
