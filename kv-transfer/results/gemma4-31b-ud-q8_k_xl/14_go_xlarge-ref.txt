### 1. Critical Bugs & Stability Issues

#### A. Potential Stack Overflow in `AssignTask`
In `scheduler.go`, the `AssignTask` method handles cancelled tasks using recursion:
```go
if task.State == TaskCancelled {
    return s.AssignTask(workerID)
}
```
**Issue:** If the priority queue contains a large number of cancelled tasks (e.g., a bulk cancellation event), this will cause a **Stack Overflow**.
**Fix:** Replace the recursion with a `for` loop.
