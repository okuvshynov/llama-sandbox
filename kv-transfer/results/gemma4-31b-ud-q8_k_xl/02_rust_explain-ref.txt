### 1. The `compress` function
This function performs the actual compression.
*   **Input:** A slice of bytes (`&[u8]`).
*   **Output:** A vector of tuples `(u8, u16)`, where the first element is the byte and the second is the count of repetitions.
*   **How it works:**
    1. It starts with the first byte of the input and sets a `count` to 1.
    2. It loops through the rest of the bytes. 
    3. If the current byte is the same as the previous one **and** the count hasn't reached the maximum limit of a `u16` (65,535), it simply increments the count.
    4. As soon as it hits a different byte, it saves the current byte and its count as a tuple `(current, count)` into the result list, then resets the counter for the new byte.
    5. After the loop ends, it pushes the final remaining pair to the list.

### 2. The `decompress` function
This function reverses the compression process.
*   **Input:** The list of tuples created by the `compress` function.
*   **Output:** The original sequence of bytes.
*   **How it works:**
    1. It iterates through every tuple `(byte, count)`.
    2. It uses a nested loop to push that specific `byte` into a new vector exactly `count` times.

### 3. The `analyze` function
This is a utility function to calculate the total frequency of every byte in the original data.
*   **Input:** A slice of bytes.
*   **Output:** A `HashMap` where the key is the byte and the value is the total number of times it appeared.
*   **How it works:**
    1. It creates an empty `HashMap`.
    2. For every byte in the input, it uses `.entry().or_insert(0)`: if the byte isn't in the map yet, it starts it at 0; then it increments the value by 1.

### 4. The `main` function (The Execution)
This is where the program puts everything together using the test string: `"AAABBBCCCCDDDDDEEEEE"`.

1.  **Processing:** It runs the `compress`, `decompress`, and `analyze` functions on the data.
2.  **Size Calculation:** 
    *   It calculates the "Compressed size" by multiplying the number of tuples by 3. 
    *   **Why 3?** Because in memory, a `u8` is 1 byte and a `u16` is 2 bytes. Together, each tuple takes 3 bytes.
3.  **Verification:** It compares the `data` slice with the `decompressed` slice to ensure the "roundtrip" was successful (meaning no data was lost).
4.  **Frequency Report:**
    *   It converts the `HashMap` from `analyze` into a `Vec`.
    *   It sorts that vector in descending order based on the count (`b.1.cmp(&a.1)`).
    *   It prints each character and how many times it appeared in the original string.
