### 1. The `compress` Function
This function takes a slice of bytes and "shrinks" it by counting consecutive duplicates.

*   **Input:** A slice of bytes `&[u8]` (e.g., `[A, A, A, B, B]`).
*   **Output:** A vector of tuples `Vec<(u8, u16)>`. Each tuple contains the byte and the number of times it appeared consecutively.
*   **The Logic:**
    1. It starts with the first byte of the input and sets the count to 1.
    2. It iterates through the rest of the bytes.
    3. **If** the current byte is the same as the previous one (and the count hasn't exceeded the maximum size of a `u16`), it simply increments the count.
    4. **Else** (if the byte changes), it saves the current byte and its count as a tuple in the `result` vector, then resets the count to 1 for the new byte.
    5. Finally, it pushes the last remaining run to the vector.

**Example:** `"AAABBBCC"` $\rightarrow$ `[('A', 3), ('B', 3), ('C', 2)]`

### 2. The `decompress` Function
This function reverses the compression process to restore the original data.

*   **Input:** A slice of tuples `&[(u8, u16)]`.
*   **Output:** The original byte sequence `Vec<u8>`.
*   **The Logic:**
    1. It iterates through every tuple `(byte, count)`.
    2. Inside that loop, it runs another loop that pushes the `byte` into the result vector exactly `count` times.

**Example:** `[('A', 3), ('B', 3)]` $\rightarrow$ `"AAABBB"`

### 3. The `analyze` Function
This function performs a frequency analysis of the input data.

*   **Input:** A slice of bytes.
*   **Output:** A `HashMap` where the **Key** is the byte and the **Value** is the total number of times that byte appears anywhere in the input.
*   **The Logic:**
    1. It creates an empty `HashMap`.
    2. It loops through every byte.
    3. The line `*freq.entry(b).or_insert(0) += 1;` is a common Rust idiom: "Find the entry for byte `b`; if it doesn't exist, insert 0; then increment the value by 1."

### 4. The `main` Function
This is where the program is executed and tested.

1.  **Data Setup:** It defines a byte string: `b"AAABBBCCCCDDDDDEEEEE"`.
2.  **Execution:** It runs the `compress`, `decompress`, and `analyze` functions.
3.  **Statistics:**
    *   **Original size:** The number of characters in the string.
    *   **Compressed entries:** How many tuples are in the compressed version.
    *   **Ratio:** It calculates the memory efficiency. It multiplies `compressed.len()` by **3** because a tuple of `(u8, u16)` takes 3 bytes (1 byte for the `u8` and 2 bytes for the `u16`).
    *   **Roundtrip OK:** It compares the original data with the decompressed data to ensure no data was lost during the process.
4.  **Frequency Print:** It converts the `HashMap` into a `Vec`, sorts it from highest frequency to lowest, and prints each character and its total count.
