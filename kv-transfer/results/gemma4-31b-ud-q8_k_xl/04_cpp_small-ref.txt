This program implements **Huffman Coding**, a famous greedy algorithm used for **lossless data compression**. The goal is to assign variable-length binary codes to characters based on their frequency: characters that appear more often get shorter codes, and characters that appear less often get longer codes.

Here is the step-by-step breakdown of the logic:

1.  **Frequency Analysis (`buildTree` part 1):**
    The program first scans the input text and counts how many times each character appears using a `std::unordered_map`.

2.  **Building the Huffman Tree (`buildTree` part 2):**
    *   It creates a "leaf node" for every unique character and puts them into a **Min-Priority Queue**. The node with the lowest frequency always stays at the top.
    *   The algorithm repeatedly removes the two nodes with the lowest frequencies, creates a new internal "parent" node with a frequency equal to the sum of the two, and pushes this parent back into the queue.
    *   This continues until only one node remains: the **Root** of the Huffman Tree.

3.  **Generating Binary Codes (`buildCodes`):**
    The program traverses the resulting tree from the root to the leaves. 
    *   Moving **left** appends a `'0'` to the code.
    *   Moving **right** appends a `'1'` to the code.
    *   When a leaf node is reached, the accumulated string of 0s and 1s becomes the unique prefix code for that character.

4.  **Encoding and Calculation (`encode` and `main`):**
    *   The program replaces every character in the original text with its new binary code.
    *   Finally, it compares the size of the original text (assuming 8 bits per character) against the size of the encoded binary string to show the compression ratio.

### Time Complexity Analysis

Let $N$ be the total number of characters in the input string and $K$ be the number of **unique** characters (the alphabet size).

#### 1. `buildTree`
*   **Frequency Counting:** Iterating through the string takes $O(N)$.
*   **Initial PQ Population:** Pushing $K$ unique characters into the priority queue takes $O(K \log K)$.
*   **Tree Construction:** The `while` loop runs $K-1$ times. Inside the loop, `pop` and `push` operations on the priority queue take $O(\log K)$. Total: $O(K \log K)$.
*   **Subtotal:** $O(N + K \log K)$.

#### 2. `buildCodes`
*   The function visits every node in the Huffman tree exactly once. A binary tree with $K$ leaves has $2K-1$ nodes.
*   **Subtotal:** $O(K)$.

#### 3. `encode`
*   The program iterates through the input string of length $N$. For each character, it performs a map lookup (average $O(1)$) and appends the code to the result string.
*   **Subtotal:** $O(N)$.

#### 4. Main Function Sorting
*   The program sorts the unique codes for display purposes. Sorting $K$ elements takes $O(K \log K)$.
*   **Subtotal:** $O(K \log K)$.
