This Java program implements an **LRU (Least Recently Used) Cache**. 

An LRU Cache is a fixed-capacity data structure that discards the least recently accessed items first when it reaches its limit. It is widely used in real-world systems (like database buffering or web servers) to keep frequently used data in fast memory while discarding old, stale data.

### The Data Structure: A Hybrid Approach
To achieve high performance, this program combines two fundamental data structures into one hybrid structure. This allows both **searching** and **reordering** to happen in constant time.

#### 1. `HashMap<K, Node<K, V>>` (The "Map")
*   **Purpose:** Provides $O(1)$ (constant time) access to any element.
*   **Role:** The Map stores the `key` as the key and a reference to the actual `Node` in the linked list as the value. This prevents us from having to "search" through a list to find an item, which would be slow ($O(n)$).

#### 2. Doubly Linked List (The "List")
*   **Purpose:** Maintains the chronological order of usage.
*   **Role:** 
    *   **The Head:** Represents the **Most Recently Used (MRU)** item.
    *   **The Tail:** Represents the **Least Recently Used (LRU)** item.
*   **Why Doubly Linked?** Each `Node` has a `prev` and `next` pointer. This allows the program to remove a node from the middle of the list in $O(1)$ time because a node "knows" who its neighbors are.
*   **Dummy Nodes:** The code uses "dummy" `head` and `tail` nodes. These are empty nodes that act as sentinels, ensuring that the list is never truly empty, which simplifies the logic by removing the need to check for `null` during every insertion or deletion.

### How the Operations Work

#### `get(K key)`
1.  Check the `HashMap` for the key.
2.  If the key doesn't exist, return `null`.
3.  If it **does** exist:
    *   The item was just used, so it is now the "Most Recently Used."
    *   The node is **removed** from its current position in the Doubly Linked List.
    *   The node is **added to the front** (just after the dummy `head`).
    *   Return the value.

#### `put(K key, V value)`
1.  **If the key already exists:**
    *   Update the existing node's value.
    *   Move the node to the front (it's now the most recently used).
2.  **If the key is new:**
    *   **Check Capacity:** If the cache is full (`map.size() >= capacity`):
        *   Identify the LRU item (the node right before the dummy `tail`).
        *   Remove that node from both the `HashMap` and the Doubly Linked List.
    *   **Create New Node:** Create a new node, add it to the `HashMap`, and insert it at the front of the list.

### Complexity Analysis

| Operation | Time Complexity | Reason |
| :--- | :--- | :--- |
| **`get(key)`** | $O(1)$ | HashMap lookup is $O(1)$; moving a node in a Doubly Linked List is $O(1)$. |
| **`put(key, val)`** | $O(1)$ | HashMap insertion/deletion is $O(1)$; adding/removing nodes from the list is $O(1)$. |
