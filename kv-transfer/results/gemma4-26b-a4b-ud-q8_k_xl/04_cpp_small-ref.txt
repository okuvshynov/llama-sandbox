This program implements **Huffman Coding**, a famous algorithm used for **lossless data compression**. 

The core idea is to assign variable-length binary codes to characters based on their frequency. Frequent characters are assigned shorter codes (e.g., `01`), while infrequent characters are assigned longer codes (e.g., `11010`). This reduces the overall number of bits required to represent a piece of text compared to standard fixed-length encoding (like ASCII, which uses 8 bits per character).

### Step-by-Step Logic

1.  **Frequency Counting (`buildTree` part 1):**
    The program first scans the input string and uses an `unordered_map` to count how many times each character appears.

2.  **Building the Huffman Tree (`buildTree` part 2):**
    *   It creates a "leaf node" for every unique character and pushes them into a **min-priority queue** (the node with the lowest frequency is always at the top).
    *   It repeatedly extracts the two nodes with the lowest frequencies.
    *   It creates a new "parent" node with a frequency equal to the sum of the two nodes.
    *   It attaches the two nodes as children to this parent and pushes the parent back into the queue.
    *   This continues until only one node remains: the **Root** of the Huffman Tree.

3.  **Generating Binary Codes (`buildCodes`):**
    The program performs a recursive Depth-First Search (DFS) traversal of the tree:
    *   Every time it moves **left**, it appends a `'0'` to the code.
    *   Every time it moves **right**, it appends a `'1'` to the code.
    *   When it reaches a leaf node (a node representing an actual character), it saves that accumulated string of 0s and 1s into a map.

4.  **Encoding and Comparison (`encode` & `main`):**
    *   The program traverses the original text and replaces every character with its new binary string.
    *   Finally, it compares the size of the new "encoded" bitstring against the size of the original text (assuming 8 bits per character) to show the compression ratio.
