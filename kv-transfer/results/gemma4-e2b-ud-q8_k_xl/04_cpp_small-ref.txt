## 1. Program Explanation

The program performs the standard steps of Huffman encoding: frequency counting, tree construction, code generation, and finally, encoding the original text.

### Key Components and Functions:

#### A. Data Structures

*   **`struct Node`**: Represents a node in the Huffman tree. It stores the character (`ch`), its frequency (`freq`), and pointers to its left and right children.
*   **`struct Compare`**: A custom comparator used by the `std::priority_queue`. It defines the ordering such that nodes with **higher** frequencies have lower priority (i.e., the smallest frequency elements bubble up to the top).

#### B. `buildTree(const std::string& text)` (Tree Construction)

This function builds the Huffman tree from the input text:

1.  **Frequency Counting:** It first iterates through the input `text` to count the occurrences of every unique character using an `std::unordered_map`.
2.  **Initialization:** It creates a `std::priority_queue` (min-heap based on frequency) and inserts a leaf `Node` for every unique character and its calculated frequency.
3.  **Tree Building:** It enters a loop that runs until only one node (the root of the complete Huffman tree) remains in the priority queue.
    *   In each iteration, it extracts the two nodes with the **lowest frequencies** (`left` and `right`).
    *   It creates a new internal `parent` node whose frequency is the sum of the two extracted nodes' frequencies.
    *   The extracted nodes become the children of the new parent.
    *   The new `parent` node is inserted back into the priority queue.

#### C. `buildCodes(Node* node, ...)` (Code Generation)

This function performs a Depth First Search (DFS) traversal on the completed Huffman tree to generate the binary codes:

*   It recursively traverses the tree. Moving to the `left` child appends `'0'` to the current prefix, and moving to the `right` child appends `'1'`.
*   When a leaf node is reached, the accumulated prefix string is assigned as the code for that character.

#### D. `encode(const std::string& text, ...)` (Encoding)

This function uses the generated `codes` map to transform the original text into the final compressed bit string:

*   It iterates through every character in the input `text`.
*   For each character, it looks up its corresponding binary code in the `codes` map and appends that code to the `result` string.

#### E. `main()`

The main function orchestrates the process:

1.  It defines the input text.
2.  It calls `buildTree` to create the optimal compression tree.
3.  It calls `buildCodes` to generate the mapping (character $\rightarrow$ binary string).
4.  It sorts the character codes based on the length of the code (shorter codes come first).
5.  It prints the generated codes for review.
6.  It calls `encode` to generate the final compressed string.
7.  It calculates and prints the compression ratio.

## 2. Time Complexity Analysis

We analyze the time complexity based on two primary variables:

*   **$N$**: The length of the input `text`.
*   **$C$**: The size of the alphabet (the number of unique characters in the text, $C \le 256$ if dealing only with ASCII).

### Analysis of `buildTree`

1.  **Frequency Counting:** Iterating through the text takes $O(N)$ time.
2.  **Initialization:** Inserting $C$ nodes into the priority queue takes $O(C \log C)$ time.
3.  **Tree Building Loop:** The loop runs $C - 1$ times. In each iteration, we perform two `pop` operations and one `push` operation on a priority queue of size at most $C$. Each heap operation takes $O(\log C)$ time.
    *   Total time for the loop: $O(C \log C)$.

**Time Complexity for `buildTree`:** $O(N + C \log C)$.

### Analysis of `buildCodes`

*   This function performs a single Depth First Search (DFS) traversal on the Huffman tree. A Huffman tree has at most $2C - 1$ nodes.
*   Since the traversal visits every node exactly once, the time taken is proportional to the number of nodes.

**Time Complexity for `buildCodes`:** $O(C)$.

### Analysis of `encode`

*   This function iterates through the original input text of length $N$.
*   For each character, it performs a map lookup (average $O(1)$) and string concatenation/append.

**Time Complexity for `encode`:** $O(N)$.

### Analysis of Sorting

*   The program extracts $C$ pairs (character, code) from the map and sorts them based on code length.
*   Sorting $C$ elements takes $O(C \log C)$ time.
