## 2. Function Breakdown

### A. `fn compress(input: &[u8]) -> Vec<(u8, u16)>`

This function performs the **Run-Length Encoding (RLE)**.

1. **Initialization:** It initializes an empty `result` vector to store the compressed pairs.
2. **Edge Case:** It checks if the input is empty and returns an empty result if so.
3. **Iteration:** It starts by setting `current` to the first byte and `count` to 1. It then iterates through the rest of the input bytes:
    * **Match Check:** If the current byte (`byte`) is the same as the `current` byte being tracked, it increments `count`. (It stops counting if `count` reaches `u16::MAX`, although this is unlikely for typical string data).
    * **Mismatch/End of Run:** If the byte is different, or if the maximum count is reached, it means the run has ended.
        * The previous `(current, count)` pair is pushed into the `result` vector.
        * The tracking is reset: `current` is updated to the new `byte`, and `count` is reset to 1.
4. **Final Push:** After the loop finishes, the last run of bytes must be recorded. It pushes the final `(current, count)` pair.
5. **Return:** Returns the vector of `(byte, count)` tuples.

### B. `fn decompress(data: &[(u8, u16)]) -> Vec<u8>`

This function reverses the RLE process.

1. **Initialization:** It initializes an empty `result` vector to hold the decompressed bytes.
2. **Iteration:** It iterates through the input slice, where each element is a tuple `(byte, count)`.
3. **Expansion:** For each tuple, it enters an inner loop that runs exactly `count` times. In each iteration of the inner loop, it pushes the `byte` into the final `result` vector.
4. **Return:** Returns the fully reconstructed vector of original bytes.
