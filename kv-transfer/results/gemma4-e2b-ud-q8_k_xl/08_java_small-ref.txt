An LRU Cache is a design pattern used to efficiently manage a fixed-size cache. The core principle is that when the cache reaches its capacity and a new item needs to be inserted, the item that has not been accessed for the longest time (the Least Recently Used item) is automatically evicted.

The `LRUCache` class allows users to store key-value pairs (`K` and `V`) in a cache with a defined maximum `capacity`.

2. **`get(K key)`:** Retrieves the value associated with the given key.
    * If the key exists, it marks that node as the Most Recently Used (MRU) by moving it to the front of the list.

3. **`put(K key, V value)`:** Inserts or updates a key-value pair.
    * **If the key already exists:** It updates the value and moves the existing node to the front (MRU).

        * It checks if the cache is full (`map.size() >= capacity`).
        * If full, it identifies the LRU element (the node just before the `tail`) and removes it from both the linked list and the map (eviction).
        * It creates a new node, adds it to the map, and places it at the front of the list (MRU position).

## 2. Data Structure Description

The `LRUCache` uses a combination of two fundamental data structures to achieve its O(1) time complexity for both `get` and `put` operations:

### A. Hash Map (`Map<K, Node<K, V>> map`)

* **Purpose:** Provides fast lookup (O(1) average time complexity) to quickly find the actual `Node` object corresponding to a given key.
* **Contents:** Maps the cache key (`K`) to the actual node object in the linked list.

### B. Doubly Linked List (Implemented via `Node` and `head`/`tail`)

* **Purpose:** Maintains the **order of usage**. This structure is crucial for identifying the Most Recently Used (MRU) item (the one linked to the `head`) and the Least Recently Used (LRU) item (the one linked to the `tail`).
* **Structure:**
    * **Nodes:** Each `Node` holds the key, value, and pointers to its `prev` and `next` nodes.
    * **Sentinel Nodes (`head` and `tail`):** Dummy nodes are used to simplify boundary checks.
        * `head.next` always points to the **MRU** element.
        * `tail.prev` always points to the **LRU** element.
* **Functionality:**
    * **Movement:** The core efficiency comes from the ability to perform **O(1) deletion and insertion**. When an item is accessed (`get` or `put`), its node is instantly removed from its current position and re-inserted at the `head` (MRU position).
    * **Eviction:** When the cache is full, the LRU item is easily identified as `tail.prev` and removed in O(1) time.

### Summary of Efficiency

By combining these two structures, the LRU Cache achieves high performance:
