## 1. Security Vulnerabilities (Critical)

### Issue 1.1: SQL Injection in Repository Layer

**Location:** `UserRepository.findByEmail`
```typescript
// VULNERABLE CODE
async findByEmail(email: string): Promise<User | null> {
    const result = await pool.query(
      "SELECT * FROM users WHERE email = '" + email + "'" // Direct string concatenation
    );
    return result.rows[0] || null;
}
```
**Problem:** Directly concatenating user input (`email`) into the SQL query string allows an attacker to inject malicious SQL commands (e.g., `' OR 1=1 --`).
**Fix:** Always use parameterized queries provided by the `pg` pool.

```typescript
// FIX
async findByEmail(email: string): Promise<User | null> {
    const result = await pool.query(
      "SELECT * FROM users WHERE email = $1", [email] // Use parameterized query
    );
    return result.rows[0] || null;
}
```

### Issue 1.2: SQL Injection in Search Service

**Location:** `SearchService.getSuggestions`
```typescript
// VULNERABLE CODE
async getSuggestions(prefix: string): Promise<string[]> {
    const result = await pool.query(
      `SELECT DISTINCT title FROM documents WHERE title ILIKE '${prefix}%' LIMIT 10` // Direct string interpolation
    );
    return result.rows.map((r: any) => r.title);
}
```
**Problem:** Similar to Issue 1.1, interpolating user input (`prefix`) directly into the query string creates a severe SQL injection vulnerability.
**Fix:** Parameterize the `LIKE` clause. PostgreSQL handles pattern matching (`%`) within parameters properly.

```typescript
// FIX
async getSuggestions(prefix: string): Promise<string[]> {
    const searchPattern = `%${prefix}%`;
    const result = await pool.query(
      `SELECT DISTINCT title FROM documents WHERE title ILIKE $1 LIMIT 10`, [searchPattern]
    );
    return result.rows.map((r: any) => r.title);
}
```

### Issue 1.3: Weak Password Hashing

**Location:** `AuthService.register` and `AuthService.login`
```typescript
// VULNERABLE CODE
const passwordHash = crypto.createHash("sha256").update(password + salt).digest("hex");
```
**Problem:** Using SHA-256 for password hashing is extremely fast and vulnerable to brute-force attacks using rainbow tables or GPU acceleration.
**Fix:** Use a dedicated, slow, adaptive hashing library like **Bcrypt**
