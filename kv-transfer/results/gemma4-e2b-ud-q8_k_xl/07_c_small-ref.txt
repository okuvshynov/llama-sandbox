### 1. Critical Bug: Memory Leak and Data Corruption in `table_delete`

The implementation of `table_delete` fails to free the memory associated with the entry being removed, leading to a severe memory leak and corrupted pointers.

**The problematic code in `table_delete`:**

```c
void table_delete(Table* t, const char* key) {
    for (int i = 0; i < t->size; i++) {
        if (strcmp(t->entries[i].key, key) == 0) {
            t->entries[i] = t->entries[t->size - 1]; // Shallow copy/overwrite
            t->size--;
            return;
        }
    }
}
```

**Why this is wrong:**

1.  **Memory Leak:** When an entry is deleted, the memory previously allocated by `strdup` for the `key` and `value` strings of the entry being removed is no longer referenced by the table structure. Since the entry is overwritten with a copy from the last element (`t->entries[t->size - 1]`), the original, now obsolete, allocated strings are **never freed**. This results in a memory leak every time an item is deleted.
2.  **Corrupted Pointers (Potential Double Free):** If the overwritten entry (`t->entries[i]`) contained pointers to dynamically allocated memory, those pointers are now stale. When `table_free` eventually runs, it will free the memory pointed to by the new entry, but the pointers from the old entry might still exist in memory, leading to potential double frees if the deletion logic is complex, or simply leaving dangling pointers if the structure relied on the old entry's memory.
