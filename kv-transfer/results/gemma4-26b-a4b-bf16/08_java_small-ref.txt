This Java program implements an **LRU (Least Recently Used) Cache**. 

An LRU cache is a fixed-capacity data structure that discards the least recently accessed items first when the capacity is reached. This is commonly used in systems where you want to keep frequently used data in fast memory (like a CPU cache or database buffer) while making room for new data.

---

### The Data Structure: A Hybrid Approach
To achieve the efficiency required for a cache (specifically $O(1)$ time complexity for both reading and writing), this program uses a **combination of two distinct data structures**:

#### 1. A Doubly Linked List
*   **Purpose:** To maintain the "order of usage."
*   **How it works:** The most recently used items are kept at the **front** (near the `head` dummy node), and the least recently used items migrate toward the **back** (near the `tail` dummy node).
*   **Why Doubly Linked?** Because each `Node` has a `prev` and `next` pointer, we can remove a node from the middle of the list in $O(1)$ time, provided we already have a reference to that node.

#### 2. A Hash Map (`HashMap`)
*   **Purpose:** To provide instant access to the nodes.
*   **How it works:** The map stores the `key` as the key and the actual `Node` object as the value.
*   **Why a Map?** Without the map, if you wanted to find a specific key, you would have to traverse the entire linked list, which would take $O(n)$ time. The map allows the program to jump directly to the correct node in $O(1)$ time.

---

### How the Methods Work

#### `get(K key)`
1.  Look up the key in the `HashMap`.
2.  If it doesn't exist, return `null`.
3.  If it **does** exist:
    *   The item has just been used, so it is no longer "least recent."
    *   **Remove** the node from its current position in the linked list.
    *   **Add** the node to the **front** of the list (making it the Most Recently Used).
    *   Return the value.

#### `put(K key, V value)`
1.  **If the key already exists:**
    *   Update its value.
    *   Move it to the **front** of the list (it is now the most recent).
2.  **If the key is new:**
    *   **Check Capacity:** If the cache is full (`map.size() >= capacity`), find the node at the very end of the list (`tail.prev`). This is the **Least Recently Used** node.
    *   **Evict:** Remove that node from both the `HashMap` and the linked list.
    *   **Insert:** Create a new node, add it to the `HashMap`, and place it at the **front** of the linked list.

### Complexity Analysis

| Operation | Time Complexity | Reason |
| :--- | :--- | :--- |
| **`get(key)`** | $O(1)$ | HashMap lookup is $O(1)$; moving a node in a Doubly Linked List is $O(1)$. |
| **`put(key, val)`** | $O(1)$ | HashMap insertion/deletion is $O(1)$; adding/removing nodes in the list is $O(1)$. |
| **Space** | $O(C)$ | Where $C$ is the capacity. We store at most $C$ nodes in both the Map and the List. |
