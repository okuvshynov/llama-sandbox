This Python program is designed to solve a **Dependency Scheduling** problem. Given a set of tasks and the order in which they must be completed (represented as directed edges), the program finds the valid sequences in which these tasks can be performed.

In computer science terms, this is the process of finding a **Topological Sort** of a **Directed Acyclic Graph (DAG)**.

The program provides two different ways to solve this:
1. **`solve`**: Finds **one** single valid execution order efficiently.
2. **`find_all`**: Finds **every possible** valid execution order using an exhaustive search.

### Detailed Algorithm Description

The program is divided into three distinct algorithmic phases:

#### 1. Graph Construction (`build_graph`)
Before any sorting can happen, the program converts the list of edges into a usable mathematical structure:
*   **Adjacency List (`graph`)**: A dictionary where each key is a task, and the value is a list of tasks that depend on it.
*   **In-Degree Map (`in_degree`)**: A dictionary that counts how many incoming dependencies each task has. A task with an in-degree of `0` is ready to be processed immediately.
*   **Node Set (`nodes`)**: A collection of all unique tasks involved.

#### 2. Finding One Valid Order (`solve` - Kahn's Algorithm)
The `solve` function implements **Kahn's Algorithm**, which is a Breadth-First Search (BFS) approach to topological sorting.

**The Steps:**
1.  **Initialize**: Identify all nodes with an `in_degree` of `0` (tasks with no dependencies) and add them to a `queue`.
2.  **Process**: While the queue is not empty:
    *   Remove a node from the queue and add it to the `result` list.
    *   For every "neighbor" (the tasks that depend on the current node), decrement their `in_degree` by 1.
    *   If a neighbor's `in_degree` reaches `0`, add it to the `queue`.
3.  **Cycle Detection**: If the final `result` list contains fewer nodes than the total number of nodes in the graph, it means a **circular dependency** (cycle) exists (e.g., A needs B, B needs A), making a valid order impossible.

**Complexity**: $O(V + E)$, where $V$ is the number of vertices (nodes) and $E$ is the number of edges.

#### 3. Finding All Valid Orders (`find_all` - Backtracking)
Since Kahn's algorithm only picks one available node at a time, it cannot find all possibilities. To find every permutation, the program uses **Backtracking** (a Depth-First Search approach).

**The Steps:**
1.  **Base Case**: If the current `path` length equals the total number of `nodes`, a complete valid sequence has been found; save it.
2.  **Recursive Step**: 
    *   Iterate through all nodes in the graph.
    *   Check if a node is **eligible**: It must not be in the current `path` AND its current `in_degree` must be `0`.
    *   **Choose**: Add the eligible node to the `path`.
    *   **Explore**: Create a copy of the current in-degrees, decrement the degrees of the current node's neighbors, and call the function recursively to pick the next node.
    *   **Un-choose (Backtrack)**: Remove the node from the `path` to reset the state and try the next available node in the loop.

**Complexity**: This is significantly more expensive than Kahn's algorithm (exponential time complexity), as it must explore many different permutations of the nodes.
