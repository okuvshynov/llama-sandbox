### The Bug
The `while` loop in `merge_sorted_lists` is designed to compare elements from both lists and append the smaller one to the `result`. However, the loop condition is `i < len(list1) and j < len(list2)`. This means the loop **stops as soon as one of the lists is exhausted**.

If one list is longer than the other, or if all the elements in one list are smaller than the remaining elements in the other, the remaining elements of the non-exhausted list are never added to the `result`.

**Example of the bug in action:**
If you called `merge_sorted_lists([1, 2], [3, 4])`:
1. `i=0, j=0`: `1 <= 3` is true $\rightarrow$ `result = [1]`, `i = 1`.
2. `i=1, j=0`: `2 <= 3` is true $\rightarrow$ `result = [1, 2]`, `i = 2`.
3. The loop condition `i < len(list1)` is now **False** (since $2 < 2$ is false).
4. The function returns `[1, 2]`, completely losing the `[3, 4]` from the second list.

*(Note: While this function is not currently called by the `running_median` logic, it is a broken component of the provided code.)*
