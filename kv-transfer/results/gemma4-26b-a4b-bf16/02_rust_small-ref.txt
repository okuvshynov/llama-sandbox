### 1. The `compress` function (Run-Length Encoding)
This function takes a slice of bytes and shrinks it by grouping identical consecutive bytes together.

*   **Input:** A slice of bytes (e.g., `[A, A, A, B, B]`).
*   **Logic:**
    1.  It initializes an empty vector called `result`.
    2.  It tracks a `current` byte and a `count` of how many times it has appeared consecutively.
    3.  It iterates through the input:
        *   If the next byte is the same as the `current` one, it increments the `count` (up to the maximum limit of a 16-bit integer, 65,535).
        *   If the next byte is different, it "packs" the current byte and its count into a tuple `(byte, count)` and adds it to the `result`. It then resets the `current` byte to the new one and resets the `count` to 1.
    4.  After the loop, it adds the final pending group to the vector.
*   **Output:** A vector of tuples. For `AAABBB`, it returns `[(65, 3), (66, 3)]` (where 65 and 66 are ASCII for A and B).

### 2. The `decompress` function
This is the inverse of the compression function. It "unpacks" the tuples back into a raw sequence of bytes.

*   **Input:** A slice of tuples `(byte, count)`.
*   **Logic:**
    1.  It creates an empty vector `result`.
    2.  It loops through every tuple in the input.
    3.  For every tuple, it runs a nested loop that pushes the `byte` into the `result` vector exactly `count` times.
*   **Output:** The original byte sequence.

### 3. The `analyze` function
This function calculates how often each unique byte appears in the input.

*   **Input:** A slice of bytes.
*   **Logic:**
    1.  It creates a `HashMap` where the **Key** is the byte and the **Value** is the number of times it has been seen.
    2.  It iterates through the input using the `.entry()` API:
        *   `entry(b)` looks for the byte.
        *   `.or_insert(0)` says: "If this byte isn't in the map yet, add it with a starting value of 0."
        *   `*freq... += 1` increments the count.
*   **Output:** A map of frequencies (e.g., `{'A': 3, 'B': 3...}`).

### 4. The `main` function (The Execution Flow)
This is where the program runs its test case.

1.  **Setup:** It defines a test string: `AAABBBCCCCDDDDDEEEEE`.
2.  **Process:**
    *   It **compresses** the data.

    *   It **analyzes** the original data to count character frequencies.
