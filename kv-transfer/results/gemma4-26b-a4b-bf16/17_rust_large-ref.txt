*Refinement on the "UB" in Deque:*
In `steal`, `let task = unsafe { std::ptr::read(task_ptr) };`.
If the owner is `push`ing, it's doing `self.buffer[idx] = Some(task);`.
In Rust, `self.buffer[idx] = ...` is a write.
Reading a `Some(task)` via `ptr::read` while a write is happening is a classic data race.
Furthermore, if `compare_exchange` fails, the code does `std::mem::forget(task)`.
Since `task` was created via `ptr::read`, it's a bitwise copy.
If the original `Option<Task>` in the buffer was `Some(task)`, we now have two `Task` objects (which contain `Arc`s) pointing to the same memory.
By forgetting the copy, we prevent the *copy* from being dropped, but we haven't actually fixed the race. The real danger is that `ptr::read` is non-atomic.
