### 1. Soundness Issues (Correctness & Safety)

#### A. Floating-Point Panics in `percentile`
In the `percentile` function, you use:
```rust
sorted.sort_by(|a, b| a.partial_cmp(b).unwrap());
```
**The Issue:** In Rust, `f64` does not implement `Ord` because `NaN` values cannot be compared. If a `NaN` (Not-a-Number) value is ever recorded into a metric (e.g., due to a calculation error elsewhere in a real app), `partial_cmp` will return `None`, and `unwrap()` will cause the entire thread to **panic**.
**The Fix:** Use `total_cmp` (available since Rust 1.62) which provides a total ordering for floats, or explicitly handle/filter `NaN` values.
```rust
sorted.sort_by(|a, b| a.total_cmp(b));
```
